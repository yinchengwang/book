# 向量索引二进制格式规范

> 版本：1.0
> 更新时间：2026-07-12

## 概述

本文档定义了向量数据库索引的二进制持久化格式，支持 HNSW、BM25 等索引类型的序列化和反序列化。

## 文件结构

```
┌─────────────────────────────────────┐
│          文件头（64 字节）            │
├─────────────────────────────────────┤
│                                     │
│          数据区（变长）               │
│                                     │
└─────────────────────────────────────┘
```

## 文件头格式（64 字节）

| 偏移 | 大小 | 字段 | 类型 | 说明 |
|------|------|------|------|------|
| 0 | 4 | magic | uint32_t | 魔数，固定为 `0x484E5357` ("HNSW") |
| 4 | 4 | version | uint32_t | 格式版本号，当前为 1 |
| 8 | 4 | dims | int32_t | 向量维度 |
| 12 | 4 | n_total | int32_t | 向量总数 |
| 16 | 4 | M | int32_t | HNSW 参数：每个节点的连接数 |
| 20 | 4 | ef_construction | int32_t | HNSW 参数：构建时搜索范围 |
| 24 | 4 | ef_search | int32_t | HNSW 参数：查询时搜索范围 |
| 28 | 4 | max_level | int32_t | HNSW 最大层数 |
| 32 | 4 | entry_pointer | int32_t | 入口节点 ID |
| 36 | 4 | metric | int32_t | 距离度量类型（0=L2, 1=IP, 2=Cosine） |
| 40 | 4 | quantization_type | int32_t | 量化类型（0=无, 1=PQ, 2=SQ） |
| 44 | 4 | code_size | int32_t | 量化编码大小 |
| 48 | 8 | created_at | int64_t | 创建时间戳（Unix 时间） |
| 56 | 8 | modified_at | int64_t | 最后修改时间戳（Unix 时间） |

**保留字段（未使用）**：
- 偏移 64-127：保留用于扩展

## 数据区格式

### HNSW 索引数据区

```
┌─────────────────────────────────────┐
│      向量数据（n_total × dims × 4）   │
├─────────────────────────────────────┤
│      层级数据（n_total × 4）          │
├─────────────────────────────────────┤
│      偏移数组（offsets_size × 4）     │
├─────────────────────────────────────┤
│      邻居数组（nb_size × 4）          │
└─────────────────────────────────────┘
```

### BM25 索引数据区

```
┌─────────────────────────────────────┐
│      文档频率表（df_map）              │
├─────────────────────────────────────┤
│      逆文档频率表（idf_map）           │
├─────────────────────────────────────┤
│      词频表（tf_maps）                │
├─────────────────────────────────────┤
│      文档长度表（doc_lengths）         │
└─────────────────────────────────────┘
```

## 向后兼容策略

1. **版本检查**：加载时检查 `version` 字段，不兼容版本返回错误
2. **迁移函数**：提供 `hnsw_persist_migrate_v0_to_v1()` 等迁移函数
3. **默认值**：新增字段使用默认值填充，确保旧文件可加载

## 校验机制

1. **魔数校验**：加载时验证 `magic == 0x484E5357`
2. **文件大小校验**：验证文件大小 ≥ 文件头大小
3. **数据一致性校验**：可选的 checksum 字段（未来版本）

## 示例

### 最小有效文件

- 文件头：64 字节
- 向量数据：0 字节（空索引）
- 总大小：64 字节

### 1000 个 128 维向量

- 文件头：64 字节
- 向量数据：1000 × 128 × 4 = 512,000 字节
- 层级数据：1000 × 4 = 4,000 字节
- 邻居数据：~200,000 字节（估计）
- 总大小：~716,064 字节
