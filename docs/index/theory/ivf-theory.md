# IVF 算法原理

## 1. 算法概述

IVF (Inverted File) 索引是一种基于聚类的近似最近邻搜索算法，源自信息检索领域的倒排索引技术。其核心思想是将向量空间划分为多个区域（桶），每个向量归属于最近的聚类中心所在的桶，搜索时只需访问查询向量最近的若干桶，从而大幅减少计算量。

### 核心思想

**空间划分 + 倒排存储**：
- **训练阶段**：使用 K-Means 聚类将向量空间划分为 `nlist` 个区域，每个区域对应一个聚类中心（质心）
- **构建阶段**：将每个向量分配到距离最近的质心所在的倒排桶（Inverted List）
- **搜索阶段**：计算查询向量与所有质心的距离，选择最近的 `nprobe` 个桶，在这些桶内进行暴力搜索

### 适用场景

- 大规模向量检索（百万级以上）
- 需要精确控制召回率与搜索速度的权衡
- 内存受限场景（可与量化压缩结合）
- 批量构建、批量查询为主的场景

### 优缺点

| 优点 | 缺点 |
|------|------|
| 内存占用可控 | 新增数据需重训练或增量训练 |
| 搜索速度可调（nprobe） | 参数敏感（nlist、nprobe） |
| 与量化技术天然兼容 | 高维数据聚类效果下降 |
| 构建速度较快 | 删除操作开销较大 |

## 2. 数据结构

### 倒排索引结构

```
                    ┌─────────────────┐
                    │   质心数组      │
                    │  centroids[]    │
                    │  [nlist × dims] │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            │                │                │
            ▼                ▼                ▼
     ┌──────────┐     ┌──────────┐     ┌──────────┐
     │ 桶 0     │     │ 桶 1     │     │ 桶 nlist-1│
     │ IDs: [1,3,7]   │ IDs: [2,5]│     │ IDs: [4,6,8,9]│
     │ Codes: [...]   │ Codes: [...]│     │ Codes: [...]│
     └──────────┘     └──────────┘     └──────────┘
```

### 核心组件

#### 2.1 质心数组（Centroids）

存储 `nlist` 个聚类中心向量：

```c
// 质心数组：nlist 个 dims 维向量
float *centroids;  // [nlist × dims]
```

质心通过 K-Means 聚类训练得到，代表向量空间的划分。

#### 2.2 倒排桶（Inverted Lists）

每个桶存储归属于该质心的向量 ID 和可选的编码：

```c
typedef struct faiss_inverted_lists {
    size_t nlist;          /* 倒排桶数量 */
    size_t code_size;      /* 单向量编码大小（0 表示不存编码） */
    int32_t **ids;         /* 每个桶的向量 ID 数组 */
    uint8_t **codes;       /* 每个桶的编码数组 */
    size_t *sizes;         /* 每个桶当前元素数 */
    size_t *caps;          /* 每个桶当前容量 */
} faiss_inverted_lists_t;
```

**设计要点**：
- 每个桶独立管理内存，支持动态扩容
- `code_size > 0` 时存储量化编码，用于快速距离计算
- ID 数组支持删除标记（墓碑位）

#### 2.3 DirectMap 映射

向量 ID 到桶位置的映射，支持 O(1) 的向量查找和删除：

```c
typedef struct faiss_direct_map {
    int32_t type;       /* 0=无映射, 1=数组映射 */
    int64_t *array;     /* array[id] = (list_no << 32) | offset */
    int32_t size;       /* 数组容量 */
} faiss_direct_map_t;
```

**Type 1 数组映射**：
- `array[id]` 直接存储向量位置信息
- 高 32 位：桶编号 `list_no`
- 低 32 位：桶内偏移 `offset`
- 查找复杂度：O(1)

### 数据流转示意

```
向量添加流程：
┌──────────┐    计算最近质心    ┌──────────┐    追加到桶    ┌──────────┐
│ 向量 x   │ ───────────────→ │ 质心 c_i │ ────────────→ │ 桶 i     │
│ id=123   │                   │ i=argmin │               │ ids=[123]│
└──────────┘                   └──────────┘               └──────────┘
                                      │
                                      ▼
                               更新 DirectMap
                               dm[123] = (i << 32) | 0
```

## 3. 核心算法

### 3.1 训练：K-Means 聚类

```
Train(vectors, n, nlist):
    1. 初始化：从向量中随机选择 nlist 个作为初始质心
    2. 迭代直到收敛或达到 max_iter：
       a. 分配：将每个向量分配到最近的质心
       b. 更新：重新计算每个簇的质心
       c. 检查：质心移动距离 < tol 则停止
    3. 返回质心数组 centroids[nlist × dims]
```

**K-Means 参数**：

| 参数 | 说明 | 典型值 |
|------|------|--------|
| n_init | 初始化次数 | 1-3 |
| max_iter | 最大迭代次数 | 20-100 |
| tol | 收敛阈值 | 1e-4 |
| random_state | 随机种子 | 固定值用于复现 |

**质心数量选择**：
- `nlist` 通常设置为 `sqrt(n)` 到 `n/1000` 之间
- 过小：每个桶太大，搜索慢
- 过大：质心存储开销大，聚类不稳定

### 3.2 添加向量

```
Add(index, vectors, n):
    for each vector v in vectors:
        1. 计算到所有质心的距离
        2. 找到最近的质心 i = argmin(dist(v, centroids))
        3. 将 (id, code) 追加到桶 i
        4. 更新 DirectMap: dm[id] = (i << 32) | offset
```

**可选的量化编码**：
- 若配置了量化器（如 PQ），将向量编码后存入 `codes` 数组
- 编码用于搜索时的快速距离近似计算

### 3.3 搜索算法

```
Search(query, k, nprobe):
    1. 质心筛选：计算 query 到所有质心的距离
    2. 选择最近的 nprobe 个质心
    3. 在选中的桶内暴力搜索：
       for each bucket in selected_buckets:
           for each vector_id in bucket:
               dist = distance(query, vectors[vector_id])
               维护 top-k 最小堆
    4. 返回 top-k 结果
```

**nprobe 参数的影响**：
- `nprobe = 1`：只搜索最近的桶，最快但召回率最低
- `nprobe = nlist`：搜索所有桶，等于暴力搜索
- 典型值：`nprobe = sqrt(nlist)` 到 `nlist/10`

### 3.4 删除操作

```
Remove(index, ids_to_remove, n):
    for each id in ids_to_remove:
        1. 通过 DirectMap 查找位置: pos = dm[id]
        2. 解析桶号和偏移: list_no = pos >> 32, offset = pos & 0xFFFFFFFF
        3. 标记删除: bucket[list_no].ids[offset] = -1 (墓碑)
        4. 清除 DirectMap: dm[id] = -1
```

**墓碑标记**：
- 删除操作不立即物理删除，而是标记为墓碑
- 搜索时跳过墓碑 ID
- 定期执行 `compact()` 真正清理空间

### 3.5 紧凑化（Compact）

```
Compact(index):
    for each bucket:
        1. 将所有墓碑后的有效元素前移
        2. 更新受影响向量的 DirectMap
        3. 释放尾部空间
```

## 4. 参数影响

### 4.1 nlist（桶数量）

| 值 | 影响 |
|-----|------|
| 过小 | 每桶向量多，搜索慢 |
| 过大 | 质心存储开销大，聚类不稳定 |
| 推荐 | `sqrt(n)` 到 `n/1000` |

### 4.2 nprobe（搜索桶数）

| 值 | 召回率 | 速度 |
|-----|--------|------|
| 1 | 低 | 最快 |
| sqrt(nlist) | 中 | 中等 |
| nlist/10 | 高 | 较慢 |
| nlist | 100% | 最慢（暴力搜索） |

**nprobe 与召回率曲线**：

```
召回率
  │
1.0├───────────────────────────────●
   │                        ┌─────┘
0.9├─────────────────┌──────┘
   │           ┌─────┘
0.8├──────┌────┘
   │ ┌────┘
0.5├─┘
   └──────────────────────────────── nprobe
     1    10    50    100    nlist
```

### 4.3 量化集成

IVF 天然支持多种量化技术：

| 量化方法 | 压缩比 | 召回率影响 | 适用场景 |
|----------|--------|------------|----------|
| 无量化 | 1× | 无 | 内存充足 |
| PQ | 8-32× | 中等 | 大规模数据 |
| SQ | 4-8× | 较低 | 快速编码 |
| OPQ | 8-32× | 较低 | 数据分布倾斜 |

**IVF-PQ 组合**：
```
┌─────────────────────────────────────────┐
│              IVF-PQ 索引                │
├─────────────────────────────────────────┤
│ 质心: nlist × dims (float32)            │
│ 桶内存储:                               │
│   - IDs: int32 数组                     │
│   - PQ Codes: uint8[m] 数组             │
│   - m = dims / subquantizers            │
└─────────────────────────────────────────┘
```

**两阶段搜索**：
1. 第一阶段：用 PQ 编码计算近似距离，筛选候选
2. 第二阶段：用原始向量重排，提高精度

## 5. 时间/空间复杂度

### 时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 训练 | O(n × nlist × iter × d) | K-Means 迭代 |
| 添加单个向量 | O(nlist × d) | 计算到所有质心距离 |
| 搜索 | O(nprobe × d + nprobe × (n/nlist) × d) | 质心筛选 + 桶内搜索 |
| 删除 | O(1) | DirectMap 定位 + 墓碑标记 |
| 紧凑化 | O(n) | 遍历所有桶 |

**搜索复杂度分析**：
- 质心筛选：`O(nlist × d)`
- 桶内搜索：`O(nprobe × (n/nlist) × d)`
- 当 `nprobe << nlist` 时，搜索接近 `O(sqrt(n) × d)`

### 空间复杂度

| 组件 | 空间复杂度 | 说明 |
|------|------------|------|
| 质心 | O(nlist × d) | float32 存储 |
| 倒排桶 | O(n) | ID 数组 |
| 量化编码 | O(n × m) | m 为 PQ 子空间数 |
| DirectMap | O(max_id) | 数组映射 |

**内存估算**：
- 无量化：`nlist × d × 4 + n × 4`（质心 + ID）
- IVF-PQ：`nlist × d × 4 + n × (4 + m)`（质心 + ID + PQ编码）

## 6. 与其他索引对比

### 总体对比

| 索引 | 搜索复杂度 | 内存占用 | 构建速度 | 增量更新 | 适用场景 |
|------|------------|----------|----------|----------|----------|
| **IVF** | O(√n × d) | 中 | 快 | 需重训练 | 大数据集、精确控制 |
| **HNSW** | O(log n) | 高 | 中 | 支持 | 高召回、动态更新 |
| **DiskANN** | O(R log n) | 低 | 慢 | Fresh 模式 | 超大规模、磁盘 |
| **LSH** | O(k log n) | 高 | 快 | 支持 | L2/内积距离 |

### IVF vs HNSW

| 维度 | IVF | HNSW |
|------|-----|------|
| 图结构 | 无图，聚类划分 | 多层图导航 |
| 内存占用 | 中（质心+桶） | 高（全图） |
| 召回率 | 依赖 nprobe | 依赖 ef_search |
| 构建时间 | 快（K-Means） | 中等 |
| 增量插入 | 需重训练或增量聚类 | 直接插入 |
| 删除 | 墓碑标记+紧凑化 | 标记删除 |
| 参数敏感度 | 中等 | 较高 |

### IVF vs DiskANN

| 维度 | IVF | DiskANN |
|------|-----|---------|
| 存储介质 | 内存（可扩展到磁盘） | SSD + 内存 PQ |
| 容量 | 受内存限制 | 十亿级 |
| 搜索延迟 | ~0.1-1ms | ~1-10ms |
| 构建速度 | 快 | 慢（两轮剪枝） |
| 量化集成 | 原生支持 PQ | 原生支持 PQ/SQ/LVQ |

### IVF 变体

| 变体 | 说明 | 特点 |
|------|------|------|
| **IVF-Flat** | 无量化，存储原始向量 | 精度最高，内存最大 |
| **IVF-PQ** | 结合 Product Quantization | 内存小，精度略降 |
| **IVF-OPQ** | 结合 Optimized PQ | 精度优于 IVF-PQ |
| **IVF-HNSW** | HNSW 替代质心筛选 | 更快的质心查找 |
| **IVF-ADC** | Asymmetric Distance Computation | 加速 PQ 距离计算 |

## 7. 实践建议

### 参数调优指南

1. **nlist 选择**：
   - 小数据集（<1M）：`nlist = sqrt(n)`
   - 中等数据集（1M-10M）：`nlist = 4096-16384`
   - 大数据集（>10M）：`nlist = 65536-262144`

2. **nprobe 调整**：
   - 从 `nprobe = sqrt(nlist)` 开始
   - 根据召回率要求逐步增大
   - 目标召回率 95%+ 通常需要 `nprobe >= nlist/10`

3. **量化选择**：
   - 内存充足：IVF-Flat
   - 内存受限：IVF-PQ，m=32-64, bits=8
   - 追求精度：IVF-OPQ

### 最佳实践

1. **训练数据量**：至少 256 × nlist 个向量用于 K-Means 训练
2. **批量插入**：避免单条插入，批量插入效率更高
3. **定期紧凑化**：删除操作后执行 compact 释放空间
4. **预热**：搜索前预加载质心到缓存
