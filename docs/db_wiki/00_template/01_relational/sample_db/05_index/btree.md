# BTree 索引

## 学习目标
- 理解 BTree/B+Tree 的结构和特性
- 掌握插入和分裂流程

## 核心概念

- **B+Tree**：平衡多路搜索树，非叶子节点只存键值
- **Page**：树中的每个节点对应一个页
- **分裂**：节点满时分裂为两个节点

## B+Tree 结构

```mermaid
graph TD
    Root[根节点<br/>10, 20] --> I1[内部节点<br/>5, 8]
    Root --> I2[内部节点<br/>15, 18]
    Root --> I3[内部节点<br/>25, 30]
    I1 --> L1[叶子: 1,3,5]
    I1 --> L2[叶子: 6,8,9]
    I2 --> L3[叶子: 12,15]
    I2 --> L4[叶子: 16,18]
    L1 -.-> L2 -.-> L3 -.-> L4
```

## 插入流程

```mermaid
flowchart TD
    A[插入键值] --> B{根节点是否为空?}
    B -->|是| C[创建根节点]
    B -->|否| D[查找插入位置]
    D --> E{节点已满?}
    E -->|否| F[直接插入]
    E -->|是| G[分裂节点]
    G --> H[将中间键提升到父节点]
    H --> I{父节点已满?}
    I -->|是| G
    I -->|否| F
```

## 分裂过程

```mermaid
graph LR
    subgraph "分裂前"
        Full["[5, 8, 10, 15, 20]<br/>(已满)"]
    end
    subgraph "分裂后"
        Left["[5, 8, 10]"]
        Right["[15, 20]"]
        Parent["父节点: 10"]
        Parent --> Left
        Parent --> Right
    end
```

## 要点总结

- B+Tree 所有数据在叶子节点，叶子节点链表连接
- 分裂向上传播，可能影响树高
