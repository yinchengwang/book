# 多模态数据库横向对比

## 分类概述

多模态数据库能够同时支持多种数据模型（KV、文档、图、时序、向量等），简化技术栈，降低数据复制开销。相比单模型数据库，多模态数据库在处理复杂业务场景时更加灵活，但需要在各模型的支持深度上做出权衡。

## 库一览

- **SurrealDB** - 融合 KV + 文档 + 图的下一代数据库，Rust 实现，多模型融合

## 与其他分类对比

| 一句话定位 | 多模态融合新范式 | 关系型标杆 | 图数据库标杆 | 文档数据库标杆 | KV 数据结构库 |

## 定位与优势

**SurrealDB 定位**：
- 一站式后端，减少技术栈复杂度
- 前后端直接通信，减少中间层
- 嵌入式部署能力，支持边缘计算
- 实时数据订阅，原生支持实时应用

**优势场景**：
- 创业公司快速 MVP（单一数据库满足多需求）
- 边缘计算（嵌入式+离线优先）
- 实时协作应用（内置订阅）
- 原型验证（schema-less + 灵活查询）

## 选型指南

- **多模型需求明确**：推荐 SurrealDB（简化架构）
- **单模型深耕需求**：选择对应分类的成熟产品
- **生产级稳定性**：建议使用成熟单模型组合

## 学习路径

1. **先学 SurrealDB** — 理解多模态概念和 SurrealQL
2. **再学单模型数据库** — 深入理解各模型的最佳实践
3. **理解分布式系统** — 为生产部署做准备
