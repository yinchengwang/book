# 边缘数据库横向对比

## 分类概述

边缘数据库专为边缘计算场景设计，将数据库部署在靠近数据源的位置，减少延迟、提升可用性、支持离线优先。它们通常具备轻量级、低资源消耗、本地优先与云同步的特点，适合 IoT、移动应用、离线应用等场景。

## 库一览

- **Turso** - 边缘分布式 SQLite，LibSQL 分支，全球低延迟访问

## 核心特性

| 维度 | Turso | 传统 SQLite | PostgreSQL |
|------|-------|-------------|------------|
| 部署位置 | 边缘/CDN | 本地/嵌入式 | 服务器/云 |
| 复制 | 多区域复制 | 无 | 主从/分布式 |
| 离线支持 | 本地优先 | 可用 | 有限 |
| 延迟 | 毫秒级(边缘) | 极低 | 取决于网络 |
| 同步 | 原生边缘同步 | 无 | 需配置 |
| 协议 | LibSQL | SQLite3 | PostgreSQL |
| 规模 | 轻量 | 轻量 | 中大型 |
| 一句话定位 | 边缘分布式 SQLite | 本地嵌入式数据库 | 企业级关系型 |

## 边缘计算场景

**Turso 架构优势**：
- 全球分布的边缘节点，数据库离用户更近
- 本地数据库优先，网络中断不影响使用
- 边缘与云端自动同步，保证数据一致性
- LibSQL 协议兼容，SQLite 生态无缝迁移

**典型使用场景**：
- **IoT 设备**：边缘网关本地存储，定期同步
- **移动应用**：离线优先，网络恢复后同步
- **零售终端**：门店本地数据库，云端汇总分析
- **内容分发**：CDN 边缘节点存储个性化数据

## 选型指南

- **边缘部署/低延迟**：推荐 Turso（全球分布）
- **离线优先应用**：推荐 Turso（本地优先架构）
- **标准服务器部署**：选择传统数据库
- **高性能 OLTP**：选择 PostgreSQL/MySQL
