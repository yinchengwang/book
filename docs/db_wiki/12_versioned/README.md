# 版本化数据库横向对比

## 分类概述

版本化数据库将 Git 版本控制思想引入数据领域，允许对数据库进行分支、提交、回滚等操作。每条数据记录都有版本历史，数据变更可审计、可回溯，特别适合数据治理、合规要求高的场景，以及需要多人协作编辑数据的应用。

## 库一览

- **Dolt** - Git 风格版本化数据库，MySQL 协议，Branch/Commit/Merge

## 与传统数据库对比

| 维度 | Dolt | MySQL | PostgreSQL |
|------|------|-------|------------|
| 版本控制 | 原生 | 无 | 无 |
| 分支 | 原生 | 无 | 无 |
| 提交历史 | 原生 | 无 | WAL 日志 |
| 回滚 | 原生 | PITR | PITR |
| 合并 | 原生 Diff/Merge | 无 | 无 |
| SQL 协议 | MySQL | 原生 | PostgreSQL |
| 审计日志 | 内置 | 需配置 | 需配置 |
| 离线克隆 | 原生 | 需导出 | 需导出 |
| 协作模式 | Git 风格 | 传统并发 | MVCC |
| 一句话定位 | Git for Databases | 传统关系型 | 功能最全关系型 |

## 核心特性

**版本控制能力**：
- `DOLT_BRANCH` — 创建和管理分支
- `DOLT_COMMIT` — 提交变更，带完整历史
- `DOLT_MERGE` — 合并分支，支持冲突处理
- `DOLT_DIFF` — 查看任意版本间差异
- `DOLT_LOG` — 查看完整提交历史
- `DOLT_CHECKOUT` — 切换分支/版本

**数据协作场景**：
- 影子表模式：生产分支 + 测试分支
- 数据验证：Pull Request 审查数据变更
- 沙箱测试：临时分支验证数据操作
- 合规审计：完整变更历史可追溯

## 选型指南

- **数据合规/审计**：推荐 Dolt（内置版本历史）
- **数据协作编辑**：推荐 Dolt（Git 协作模式）
- **高并发 OLTP**：选择传统关系型数据库
- **生产数据存储**：建议使用成熟数据库，Dolt 可用于数据仓库场景

## 学习路径

1. **先学 Git 基础** — 分支、提交、合并概念
2. **再学 Dolt** — 理解数据库版本的类比
3. **然后学 SQL** — 掌握数据操作基础
4. **探索 DoltHub** — 了解数据分享社区
