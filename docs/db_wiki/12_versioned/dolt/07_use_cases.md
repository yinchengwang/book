# Dolt 使用场景与选型对比

## 学习目标

- 理解 Dolt 的最佳适用场景
- 掌握数据版本管理方法

## 适用场景

```mermaid
graph TD
    A[Dolt 适用场景] --> B[数据版本管理]
    A --> C[CI/CD 数据库变更]
    A --> D[数据协作]
    A --> E[审计追踪]

    B --> B1[追踪数据变更历史]
    B --> B2[回滚到任意版本]

    C --> C1[Schema 变更管理]
    C --> C2[测试数据管理]

    D --> D1[多人编辑数据]
    D --> D2[合并团队分支]
```

## 选型对比

| 维度 | Dolt | Git-LFS | Liquibase | 传统备份 |
|------|------|---------|-----------|----------|
| 数据版本控制 | 原生 | 文件级 | Schema 级 | 全量 |
| 分支管理 | 支持 | 不支持 | 不支持 | 不支持 |
| 冲突解决 | 行级 | 文件级 | 手动 | 无 |
| SQL 查询 | 支持 | 不支持 | 不支持 | 不支持 |
| 时间旅行 | 支持 | 不支持 | 不支持 | 不支持 |

## 要点总结

- **数据版本管理**是 Dolt 的核心场景
- **CI/CD 集成**可自动化数据库变更流程
- 比 Git-LFS 更精细的数据版本控制
