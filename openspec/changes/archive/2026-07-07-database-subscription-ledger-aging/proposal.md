# 数据库订阅、对账与老化机制

## Summary

实现数据库的三大扩展机制：
1. **订阅 (Subscription)**：CDC 变更数据捕获，支持实时订阅数据库变更
2. **对账 (Ledger)**：账务一致性校验，支持分布式事务的幂等性保障
3. **老化 (Aging)**：冷热数据分层管理，自动清理过期数据

## Why

现代数据库系统需要支持：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        业务场景需求                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  订阅场景：                                                          │
│  ├─ 微服务间数据同步                                                │
│  ├─ 实时数据仓库填充                                                │
│  ├─ 搜索索引实时更新                                                │
│  └─ 审计日志实时记录                                                │
│                                                                     │
│  对账场景：                                                          │
│  ├─ 分布式事务一致性校验                                            │
│  ├─ 账务系统流水核对                                                │
│  ├─ 数据同步完整性验证                                              │
│  └─ 幂等操作去重                                                    │
│                                                                     │
│  老化场景：                                                          │
│  ├─ 日志数据自动清理                                                │
│  ├─ 历史数据归档存储                                                │
│  ├─ 内存热点数据管理                                                │
│  └─ 存储空间回收                                                    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## Goals

### Phase 1: 订阅模块 ✅
- [x] CDC 变更捕获接口设计
- [x] WAL 日志解析器
- [x] 订阅管理器（支持多订阅者）
- [ ] 消息队列集成（可选）

### Phase 2: 对账模块 ✅
- [x] 账本数据结构设计
- [x] 账目录入与查询
- [x] 一致性校验算法
- [x] 幂等性保障机制

### Phase 3: 老化模块 ✅
- [x] 老化策略定义（TTL、访问频率、容量）
- [x] 冷热数据分层
- [x] 自动清理调度器
- [x] 归档存储支持

## Scope

### In Scope
- 订阅模块完整实现
- 对账模块核心功能
- 老化模块策略框架

### Out of Scope
- 分布式事务（2PC/TCC）
- 消息队列具体集成（Kafka/RabbitMQ）
- 多租户隔离

## Risks

| 风险 | 影响 | 缓解 |
|------|------|------|
| CDC 解析性能 | 高并发下解析成为瓶颈 | 异步处理 + 批量提交 |
| 账本存储膨胀 | 长期运行后账本过大 | 分区存储 + 压缩 |
| 老化误删数据 | 重要历史数据被清理 | 多级确认 + 归档优先 |
