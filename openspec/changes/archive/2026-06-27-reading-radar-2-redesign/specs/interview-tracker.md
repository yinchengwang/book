# Interview Tracker 面试追踪

## Overview

面试追踪器帮助管理求职流程，记录面试进度、面经和复盘。

### F10.3 面试状态机

**描述**：标准面试流程

**状态定义**：
```
┌─────────────────────────────────────────────────────┐
│                                                     │
│ [1] [2] [3] [4] [5] [6] [7]                      │
│ 投递  笔试  一面  二面  三面  HR   Offer         │
│                                                     │
│                                                      │
│ [8] [9] [10] [11]                                 │
│ 体检 背调  入职  归档                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**状态详情**：
| 状态 | 说明 | 可添加内容 |
|------|------|------------|
| 投递 | 简历已投 | 岗位JD、公司信息 |
| 笔试 | 在线测评 | 笔试时间、题目回忆 |
| 一面 | 技术面 | 时间、面试官、题目、反馈 |
| 二面 | 技术面/交叉面 | 同上 |
| 三面 | 高管面 | 同上 |
| HR | hr面 | 薪资谈判 |
| Offer | offer | 薪资、福利、截止日期 |
| 体检 | 入职体检 | 体检结果 |
| 背调 | 背景调查 | 进度 |
| 入职 | 入职日期 | 入职准备 |
| 归档 | 流程结束 | 最终状态 |

### F10.4 面试记录

**描述**：每次面试的详细记录

**记录模板**：
```typescript
interface InterviewRecord {
  id: string;
  companyId: string;
  stage: InterviewStage;
  date: string;
  time?: string;
  interviewer?: string;      // 面试官
  type?: '技术' | 'HR' | '高管' | '交叉';
  duration?: number;          // 分钟
  questions?: string[];      // 面试题目
  answers?: string[];        // 我的回答
  feedback?: string;         // 面试反馈
  selfReview?: string;       // 自我复盘
  rating?: 1 | 2 | 3 | 4 | 5; // 自我评分
  status: 'pending' | 'passed' | 'failed' | 'unknown';
}
```

```
个人总结：
- 算法题刷了 LeetCode 经典题
- 项目要有亮点，深挖项目细节

#字节跳动 #面经 #后端
```

### F10.6 统计面板

**描述**：面试数据统计

**指标**：
| 指标 | 计算 |
|------|------|
|投递总数| 所有公司计数|
|面试中| 状态在笔试~HR 之间的 |
|已Offer| Offer 状态计数 |
|Offer率| Offer / 面试数 |
|平均周期| 从投递到 Offer 的平均天数 |
|最热公司| 投递最多的公司 |

**可视化**：
- Offer 率环形图
- 面试周期趋势图
- 公司投递热力图

## Data Models

```typescript
// 公司
interface Company {
  id: string;
  name: string;
  position: string;
  location?: string;
  salary?: string;
  hrContact?: string;
  hrName?: string;
  notes?: string;
  stage: InterviewStage;
  appliedAt: string;
  updatedAt: string;
}

// 面试轮次
interface InterviewRound {
  id: string;
  companyId: string;
  stage: InterviewStage;
  date: string;
  interviewer?: string;
  feedback?: string;
  questions?: string[];
  selfReview?: string;
  rating?: 1 | 2 | 3 | 4 | 5;
  status: 'pending' | 'passed' | 'failed' | 'unknown';
}

// 面试状态枚举
type InterviewStage = 
  | 'applied'      // 投递
  | 'test'         // 笔试
  | 'first'        // 一面
  | 'second'       // 二面
  | 'third'        // 三面
  | 'hr'           // HR面
  | 'offer'        // Offer
  | 'physical'     // 体检
  | 'background'   // 背调
  | 'onboard'      // 入职
  | 'archived';    // 归档
```

## Technical Notes

- 状态机：基于状态机模式管理
- 时间线：垂直时间轴 UI
- 提醒：面试前提醒（Web Notifications）
- 导出：面经导出为 Markdown
