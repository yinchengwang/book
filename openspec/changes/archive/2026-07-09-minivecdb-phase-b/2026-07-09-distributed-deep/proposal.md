## 技术方案

### 分布式 MVCC

```
┌─────────────────────────────────────────────────────────────────┐
│                    分布式 MVCC 架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│   Client ─→ Coordinator                                          │
│               │                                                  │
│               ▼                                                  │
│         ┌───────────┐                                           │
│         │ GTX Alloc │ (全局事务 ID 分配)                        │
│         └───────────┘                                           │
│               │                                                  │
│         ┌─────┴─────┐                                           │
│         ▼           ▼                                           │
│      Node A       Node B                                        │
│    ┌──────┐     ┌──────┐                                       │
│    │ xmin │     │ xmin │  ← 本地事务 ID                        │
│    │ gtxid│     │ gtxid│  ← 全局事务 ID                        │
│    └──────┘     └──────┘                                       │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 故障转移流程

```
1. 心跳检测 → 发现节点失联
2. 触发选举 → Raft 选新 Leader
3. 状态同步 → 新 Leader 补全数据
4. 服务注册 → 更新路由表
5. 流量切换 → 负载均衡器更新
```
