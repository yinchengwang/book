# B1: 存储引擎深度

## 概述

深化存储引擎能力，实现 MVCC、完整检查点、死锁检测等生产级特性。

## 问题背景

当前存储引擎具备基础能力，但在生产环境中需要：
- 多版本并发控制 (MVCC) 保证隔离性
- 完整的崩溃恢复机制
- 死锁检测避免系统挂起
- 事务回滚保证原子性

## 目标

- MVCC 隔离级别可配置
- 完整检查点与崩溃恢复
- 死锁检测与处理
- 事务回滚机制完善

## 技术方案

### MVCC 版本链

```
┌─────────────────────────────────────────────────────────────┐
│                        Tuple 版本链                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐              │
│   │ Version3│←────│ Version2│←────│ Version1│ (最新)     │
│   │ xmin=3  │     │ xmin=2  │     │ xmin=1  │             │
│   │ xmax= - │     │ xmax=4  │     │ xmax= - │             │
│   └─────────┘     └─────────┘     └─────────┘              │
│                                                             │
│   Heap Page Header:                                        │
│     - pd_xid_base: 基础事务 ID                             │
│     - pd_multixact_offset: 多事务偏移                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 检查点结构

```
┌─────────────────────────────────────────────────────────────┐
│                    Checkpoint Record                        │
├─────────────────────────────────────────────────────────────┤
│   CheckpointHeader:                                        │
│     - magic_number                                        │
│     - version                                             │
│     - checkpoint_loc (下一个检查点位置)                   │
│     - prev_checkpoint_loc                                 │
│     - xlog_rec_ptr (REDO 点)                             │
│     - next_xid (下一个自由事务 ID)                        │
│     - next_oid (下一个自由 OID)                           │
├─────────────────────────────────────────────────────────────┤
│   FullPageImage:                                          │
│     - 每个脏页的第一个完整镜像                             │
└─────────────────────────────────────────────────────────────┘
```
