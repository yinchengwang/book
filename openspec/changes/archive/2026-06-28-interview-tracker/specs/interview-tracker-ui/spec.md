# 面试追踪 UI 规范

## ADDED Requirements

### Requirement: 公司卡片网格视图
系统必须以矩形卡片形式展示公司记录，多个卡片按横向平铺排列，支持横向滚动。

#### Scenario: 默认展示公司卡片
- **WHEN** 页面加载且有公司数据
- **THEN** 系统渲染公司卡片网格，每张卡片显示公司名、岗位名、地域、薪资、技术栈、方向、状态进度条和更新时间

#### Scenario: 卡片点击打开详情
- **WHEN** 用户点击任意公司卡片
- **THEN** 系统从底部弹出详情面板（Sheet），展示该公司的完整信息

#### Scenario: 卡片横向溢出时显示滚动
- **WHEN** 公司卡片总数超出可视宽度
- **THEN** 容器显示横向滚动条，卡片保持固定宽度不被压缩

### Requirement: 时间线分组展示
系统必须按月份对公司卡片进行分组展示，形成时间线结构。

#### Scenario: 按月份分组
- **WHEN** 页面渲染公司列表
- **THEN** 系统根据每个公司最后一次状态更新的日期，将其归入对应的月份组（如"2026年6月"）

#### Scenario: 月份组可折叠
- **WHEN** 用户点击月份组标题
- **THEN** 系统折叠/展开该月份下的所有公司卡片

#### Scenario: 状态变更时卡片跨月移动
- **WHEN** 用户更新了某公司的状态（状态变更）
- **THEN** 系统将该卡片移动到对应新月份组的末尾（不变更状态的内容修改不触发移动）

### Requirement: 状态进度条
每张公司卡片顶部必须显示状态进度条，直观展示当前所处阶段。

#### Scenario: 进度条圆点渲染
- **WHEN** 渲染公司卡片
- **THEN** 系统根据 14 种状态定义，用圆点序列表示进度，当前状态及之前的圆点为实心填充，之后为空心轮廓

#### Scenario: 当前状态高亮
- **WHEN** 渲染公司卡片
- **THEN** 当前所处状态的圆点加粗显示，颜色为该状态的组别颜色

#### Scenario: Offer 状态特殊标记
- **WHEN** 公司状态为 Offer
- **THEN** 进度条中最后一个圆点显示 ★ 符号而非普通圆点

### Requirement: 详情面板（底部 Sheet）
点击公司卡片后必须从底部弹出详情面板，包含多个 Tab。

#### Scenario: 详情面板打开
- **WHEN** 用户点击公司卡片
- **THEN** 系统展示底部弹出面板，覆盖屏幕下半部分，显示公司基本信息和 Tab 栏

#### Scenario: Tab 切换
- **WHEN** 用户在详情面板中点击不同 Tab（JD / 技术问题 / 综合面 / 备注 / 设置）
- **THEN** 系统切换显示对应 Tab 的内容区域

#### Scenario: 关闭详情面板
- **WHEN** 用户点击关闭按钮或遮罩层
- **THEN** 系统关闭详情面板，恢复公司卡片视图

### Requirement: 新建弹窗
系统必须提供新建公司记录的弹窗表单。

#### Scenario: 打开新建弹窗
- **WHEN** 用户点击"[+ 新建面试]"按钮
- **THEN** 系统弹出新建表单，包含公司名、岗位名、技术栈、方向、Base 地域、薪资（月薪×月数）、初始状态等字段

#### Scenario: 年包实时计算
- **WHEN** 用户在薪资字段中输入月薪和月数
- **THEN** 系统实时计算并显示年包金额（月薪 × 月数），单位万元

#### Scenario: 提交新建记录
- **WHEN** 用户填写完表单并点击"创建"
- **THEN** 系统创建公司记录（在 Markdown 文件系统中生成对应文件），刷新卡片列表，关闭弹窗

### Requirement: 月份选择器
系统必须提供月份下拉选择器，允许用户筛选特定月份。

#### Scenario: 选择特定月份
- **WHEN** 用户从月份下拉框选择一个月份
- **THEN** 系统仅显示该月份的公司卡片，隐藏其他月份

#### Scenario: 选择全部月份
- **WHEN** 用户选择"全部月份"
- **THEN** 系统显示所有月份的公司卡片

### Requirement: 地域过滤
系统必须提供地域多选过滤功能。

#### Scenario: 选择单一地域
- **WHEN** 用户在过滤面板中选择"北京"
- **THEN** 系统仅显示 Base 为北京的公司

#### Scenario: 选择多个地域
- **WHEN** 用户同时选择"北京"和"上海"
- **THEN** 系统显示 Base 为北京或上海的所有公司

#### Scenario: 选择全部地域
- **WHEN** 用户选择"全部"或取消所有勾选
- **THEN** 系统显示所有公司（地域过滤不生效）

### Requirement: 关键词搜索
系统必须支持通过关键词搜索公司名、岗位名和技术栈。

#### Scenario: 输入关键词搜索
- **WHEN** 用户在搜索框中输入"字节"
- **THEN** 系统仅显示公司名或岗位名中包含"字节"的公司

#### Scenario: 搜索无结果
- **WHEN** 用户搜索没有匹配的公司
- **THEN** 系统显示空状态提示"暂无匹配结果"

### Requirement: 排序功能
系统必须提供多种排序方式。

#### Scenario: 按最新状态更新排序
- **WHEN** 用户选择"最新状态更新"排序
- **THEN** 系统按公司最后状态更新日期降序排列

#### Scenario: 按最新创建时间排序
- **WHEN** 用户选择"最新创建时间"排序
- **THEN** 系统按公司创建日期降序排列

#### Scenario: 按薪资降序排序
- **WHEN** 用户选择"薪资降序"排序
- **THEN** 系统按年包金额从高到低排列

### Requirement: 快捷筛选条
系统必须在工具栏下方显示快捷筛选条。

#### Scenario: 点击"流程中"筛选
- **WHEN** 用户点击快捷筛选条中的"流程中"
- **THEN** 系统仅显示状态为意向公司/沟通中/已投递/简历筛选/一面/二面/三面/主管面/HR面/薪资谈判/背调 的公司

#### Scenario: 点击"Offer"筛选
- **WHEN** 用户点击快捷筛选条中的"Offer"
- **THEN** 系统仅显示状态为 Offer 的公司

#### Scenario: 点击"已拒绝"筛选
- **WHEN** 用户点击快捷筛选条中的"已拒绝"
- **THEN** 系统仅显示状态为已拒绝的公司

### Requirement: 统计概览
系统必须在页面顶部显示统计概览栏。

#### Scenario: 统计数字准确
- **WHEN** 公司有 N 条记录
- **THEN** 统计概览显示：总计 N、流程中 M1、待回复 M2、Offer M3、已拒绝 M4，各项数字之和等于总计

#### Scenario: 状态变更后统计更新
- **WHEN** 用户变更某公司的状态
- **THEN** 统计概览数字实时更新

### Requirement: 响应式布局
系统必须适配桌面端、平板和手机端三种视口。

#### Scenario: 桌面端布局（>768px）
- **WHEN** 屏幕宽度大于 768px
- **THEN** 工具栏横向排列，卡片网格显示 3-4 列

#### Scenario: 平板布局（481-768px）
- **WHEN** 屏幕宽度在 481-768px 之间
- **THEN** 工具栏换行排列，卡片网格显示 2 列

#### Scenario: 手机端布局（<=480px）
- **WHEN** 屏幕宽度小于等于 480px
- **THEN** 工具栏折叠为汉堡菜单，卡片列表单列全宽显示，详情面板全屏覆盖
