# 大变更: MiniVecDB 端到端系统

## 概述

将项目从"组件孤岛"演进为"端到端可用的向量数据库系统"。

## 背景

项目当前状态：
- ✅ 向量索引生态完备 (HNSW/DiskANN/IVF/BM25/ReRanker)
- ✅ 分布式组件完备 (Phase 9 成果)
- ✅ 多模态存储接口完备
- ⚠️ 向量层与存储层未对接
- ⚠️ 缺乏统一 API 层
- ⚠️ 缺乏端到端集成测试

## 目标

构建一个真正可用的向量数据库系统，支持：
1. 集合 CRUD 操作
2. 向量插入、搜索、删除
3. 持久化存储
4. 分布式部署
5. 端到端可测试

## 变更结构

本大变更包含以下小变更：

### Phase A: 串联上线

| 变更 | 名称 | 描述 |
|------|------|------|
| A1 | `2026-07-09-vector-persist-layer` | 向量索引持久化层 |
| A2 | `2026-07-09-vector-executor` | 向量查询执行器 |
| A3 | `2026-07-09-api-cli-layer` | API 层 + CLI 工具 |
| A4 | `2026-07-09-integration-test` | 端到端集成测试 |

### Phase B: 深化打磨 (后续)

| 变更 | 目录 | 描述 |
|------|------|------|
| B1 | `2026-07-09-storage-engine-deep` | 存储引擎深度 (MVCC/检查点/死锁) |
| B2 | `2026-07-09-query-optimizer-deep` | 查询优化器深度 (代价模型/索引选择) |
| B3 | `2026-07-09-distributed-deep` | 分布式系统深度 (分布式MVCC/故障转移) |

### Phase C: 亮剑证明 (后期)

| 变更 | 目录 | 描述 |
|------|------|------|
| C1 | `2026-07-09-benchmark-suite` | Benchmark 基准测试 |
| C2 | `2026-07-09-tech-docs` | 技术文档输出 |

## 执行顺序

```
A1 → A2 → A3 → A4 → Phase A 完成
                    ↓
              C1 (可并行)
                    ↓
              Phase B (B1 → B2 → B3)
                    ↓
              Phase C (C1 → C2)
```
