# ART 索引规格

## 功能概述

Adaptive Radix Tree（自适应基数树）索引，复用项目已有实现。

## 核心概念

### ART 简介

- 前缀压缩的 Trie 变体
- 自适应节点大小（4、16、48、256子节点）
- 内存效率高，查询性能好
- 支持字符串键

### 节点类型

```
┌─────────────────────────────────────────────────────────────┐
│                      节点类型层级                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  N4 (4 children)                                            │
│      ↓ 节点满时扩展                                         │
│  N16 (16 children)                                          │
│      ↓ 节点满时扩展                                         │
│  N48 (48 children)                                          │
│      ↓ 节点满时扩展                                         │
│  N256 (256 children)                                        │
│                                                             │
│  压缩路径节点:                                               │
│  N8 (8 children, 不常用)                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 适配器接口

为了与存储引擎集成，需要创建适配器层：

```c
// ART 索引句柄
typedef struct art_index_s art_index_t;

// 创建索引
art_index_t *art_create(storage_t *storage);

// 打开已有索引
art_index_t *art_open(storage_t *storage, page_id_t root_page_id);

// 获取根页面
page_id_t art_get_root(art_index_t *index);

// 插入
int art_insert(art_index_t *index, const void *key, size_t key_len, 
               page_id_t value_page_id);

// 查找
int art_lookup(art_index_t *index, const void *key, size_t key_len,
               page_id_t *out_value);

// 范围查询
int art_scan(art_index_t *index, 
             const void *start_key, size_t start_len,
             const void *end_key, size_t end_len,
             art_callback_t callback, void *ctx);

// 删除
int art_delete(art_index_t *index, const void *key, size_t key_len);
```

## 设计决策

1. **持久化方式**
   - ART 节点存储在数据页中
   - 根节点页面 ID 记录在元数据中
   - 崩溃后通过根节点重建索引

2. **并发控制**
   - 简单方案：写锁保护整个索引
   - 进阶方案：乐观锁 + 版本号

3. **内存 vs 磁盘**
   - 热节点保留在缓存池
   - 冷节点自动淘汰到磁盘
