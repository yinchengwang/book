# 事务与 WAL 规格

## 功能概述

基于预写日志（WAL）的事务系统，提供 ACID 保证。

## 核心概念

### ACID 特性

| 特性 | 说明 | 实现方式 |
|------|------|----------|
| Atomicity | 原子性 | WAL 日志 |
| Consistency | 一致性 | 约束检查 |
| Isolation | 隔离性 | 锁/MVCC |
| Durability | 持久性 | WAL + 刷盘 |

### WAL 原理

```
┌─────────────────────────────────────────────────────────────────┐
│                      Write-Ahead Log                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   事务 T1:                                                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │ BEGIN T1                                                │  │
│   │ UPDATE page_5: "A" → "B"                                │  │
│   │ UPDATE page_8: "X" → "Y"                                │  │
│   │ COMMIT T1                                               │  │
│   └─────────────────────────────────────────────────────────┘  │
│                           │                                    │
│                           ▼                                    │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  日志写入磁盘（先于数据）                                 │  │
│   └─────────────────────────────────────────────────────────┘  │
│                           │                                    │
│                           ▼                                    │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  数据写入磁盘                                           │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   如果崩溃发生在 A 点 → 从 WAL 恢复数据                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 接口设计

### 事务控制

```c
// 事务句柄
typedef struct txn_s txn_t;

// 开始事务
txn_t *txn_begin(db_t *db);

// 提交事务
int txn_commit(txn_t *txn);

// 回滚事务
int txn_rollback(txn_t *txn);

// 释放事务资源
void txn_free(txn_t *txn);
```

### 事务内操作

```c
// 在事务内执行 KV 操作
int txn_put(txn_t *txn, const void *key, size_t key_len,
            const void *value, size_t value_len);

int txn_get(txn_t *txn, const void *key, size_t key_len,
            void **out_value, size_t *out_len);

int txn_delete(txn_t *txn, const void *key, size_t key_len);
```

## WAL 日志格式

### 日志文件结构

```
┌─────────────────────────────────────────────────────────────────┐
│                     WAL 文件格式                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ Header (32 bytes)                                         │ │
│  │   magic: 0x57414C00 (4B)                                  │ │
│  │   version: uint32 (4B)                                    │ │
│  │   db_id: uint64 (8B)                                      │ │
│  │   start_lsn: uint64 (8B)                                  │ │
│  │   end_lsn: uint64 (8B)                                    │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ Log Record 1                                              │ │
│  │   lsn: uint64 | size: uint32 | type: uint8                │ │
│  │   txn_id: uint64 | prev_lsn: uint64                       │ │
│  │   data: ...                                               │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐ │
│  │ Log Record 2...                                           │ │
│  └───────────────────────────────────────────────────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 日志记录类型

| 类型 | 值 | 说明 |
|------|-----|------|
| LOG_BEGIN | 1 | 事务开始 |
| LOG_UPDATE | 2 | 页面更新 |
| LOG_INSERT | 3 | 插入记录 |
| LOG_DELETE | 4 | 删除记录 |
| LOG_COMMIT | 5 | 事务提交 |
| LOG_ROLLBACK | 6 | 事务回滚 |
| LOG_CHECKPOINT | 7 | 检查点 |

### 更新日志格式

```
┌─────────────────────────────────────────────────────────────┐
│ UPDATE 日志                                                  │
├─────────────────────────────────────────────────────────────┤
│ page_id: uint32      - 被更新的页面 ID                       │
│ offset: uint16       - 在页面内的偏移                        │
│ old_data: bytes      - 旧数据（用于回滚）                    │
│ new_data: bytes      - 新数据                               │
└─────────────────────────────────────────────────────────────┘
```

## 崩溃恢复

### 恢复流程

```
┌─────────────────────────────────────────────────────────────┐
│                      崩溃恢复流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 分析阶段（Analysis）                                    │
│     ├── 读取 WAL 文件                                       │
│     ├── 确定崩溃时的活动事务                                │
│     └── 确定需要重做的已提交事务                            │
│                                                             │
│  2. 重做阶段（Redo）                                        │
│     ├── 从检查点开始                                        │
│     ├── 重做所有已提交事务的操作                            │
│     └── 跳过未提交事务的操作                                │
│                                                             │
│  3. 回滚阶段（Undo）                                        │
│     ├── 对未提交事务                                         │
│     ├── 应用 UNDO 日志                                       │
│     └── 标记事务为回滚                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 检查点（Checkpoint）

定期创建检查点，减少恢复时间：

```c
// 创建检查点
int checkpoint_create(db_t *db);

// 检查点包含：
// - 所有脏页刷盘
// - WAL 日志截断
// - 元数据更新
```
