# VDB 核心索引算法深度内容

## Purpose

为向量数据库核心索引算法提供深度技术文章，涵盖 Flat 精确搜索、IVF 家族、HNSW 及图索引族谱等核心知识点，每篇文章遵循严格的 8 段式模版，包含原理分析、复杂度权衡、代码实现、工程优化、生产实践及论文溯源。

## Requirements

### Requirement: VDB 核心索引算法深度文章

VDB 核心索引算法的每篇深度文章 SHALL 覆盖以下知识点：

| 知识点 ID | 标题 | 难度 |
|-----------|------|------|
| `db-vector-basic` | Flat 精确搜索 | basic |
| `db-ivf-variants` | IVF 家族 | intermediate |
| `db-hnsw` | HNSW | intermediate |
| `db-graph-index` | 图索引族谱 | advanced |

每篇文章 SHALL 遵循 8 段式模版：

1. **引子** — 直觉比喻、解决什么问题、学习动机
2. **核心原理** — 算法流程（含手算/图解）、数学直觉、输入/输出定义
3. **复杂度分析** — 时间/空间/精度//召回权衡表
4. **代码说话** — 不少于 2 种语言实现（Python Faiss API + 至少一种 C++/Python 手写版本），代码必须可运行
5. **工程优化技术** — SIMD/内存布局/多线程/参数调优等 Faiss 级工程细节
6. **生产实践 + 决策指南** — 何时用此索引、何时换其他、反直觉场景（如强过滤回退 Flat）
7. **论文溯源** — 原始论文信息（标题/作者/会议/年份/引用量/arXiv 链接）、核心贡献一句话、算法演化谱系（前身/变体/后继）
8. **面试必考点 + 参考** — 高频问题速查表 + 扩展阅读链接

#### Scenario: 文章完成度检查

- **WHEN** 用户打开 learn.html 中对应的知识点
- **THEN** 系统 SHALL 显示完整的 8 段内容，代码示例可复制，参考链接可达

#### Scenario: 论文溯源完整性

- **WHEN** 用户阅读第 7 节论文溯源
- **THEN** 内容 SHALL 包含论文标题、作者、发表会议/期刊、年份、引用量级、arXiv 链接
