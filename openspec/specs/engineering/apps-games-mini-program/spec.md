# apps-games-mini-program

游戏小程序统一入口规范，支持三个游戏的集成运行。

## Purpose

该规格定义了 apps/games/mini_program 模块的功能需求，作为 PC 终端版游戏的统一入口。

## Requirements

### Requirement: 主菜单显示
系统 SHALL 显示游戏列表菜单，让用户选择要运行的游戏。

#### Scenario: 显示游戏选择菜单
- **WHEN** 用户启动游戏小程序
- **THEN** 显示带装饰边框的游戏列表：
  ```
  ╔═══════════════════════════════╗
  ║       🎮 游戏中心 🎮           ║
  ╠═══════════════════════════════╣
  ║  1. 贪吃蛇                     ║
  ║  2. 数独                       ║
  ║  3. 2048                       ║
  ║  0. 退出                       ║
  ╚═══════════════════════════════╝
  请选择游戏 [0-3]:
  ```

#### Scenario: 选择贪吃蛇
- **WHEN** 用户输入 1
- **THEN** 启动贪吃蛇游戏，游戏结束后返回主菜单

#### Scenario: 选择数独
- **WHEN** 用户输入 2
- **THEN** 启动数独游戏，游戏结束后返回主菜单

#### Scenario: 选择 2048
- **WHEN** 用户输入 3
- **THEN** 启动 2048 游戏，游戏结束后返回主菜单

#### Scenario: 选择退出
- **WHEN** 用户输入 0
- **THEN** 显示"再见！"并退出程序

### Requirement: 游戏返回机制
系统 SHALL 在游戏结束后返回主菜单，而不是直接退出。

#### Scenario: 游戏结束后返回
- **WHEN** 任何游戏结束并按任意键
- **THEN** 返回主菜单显示游戏列表

### Requirement: 难度选择
系统 SHALL 在启动每个游戏前显示难度选择菜单。

#### Scenario: 贪吃蛇难度选择
- **WHEN** 选择贪吃蛇后
- **THEN** 显示：
  ```
  1. 简单 (速度 180ms)
  2. 困难 (速度 120ms)
  3. 地狱 (速度 80ms)
  ```

#### Scenario: 数独难度选择
- **WHEN** 选择数独后
- **THEN** 显示：
  ```
  1. 简单 (30 个空格)
  2. 中等 (40 个空格)
  3. 困难 (50 个空格)
  ```

#### Scenario: 2048 难度选择
- **WHEN** 选择 2048 后
- **THEN** 显示：
  ```
  1. 入门 (初始 1 个块)
  2. 简单 (初始 2 个块)
  3. 困难 (初始 3 个块)
  ```
