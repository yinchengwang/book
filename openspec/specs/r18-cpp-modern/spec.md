# R18 C++ 现代特性能力规格

## 概述

R18 C++ 现代特性能力涵盖 C++11/14/17/20 的核心语言特性，为后续系统编程和工程实践奠定基础。

## 能力清单

### C++11 核心特性

#### C++11-Smart-Pointer
- **描述**: 智能指针（unique_ptr/shared_ptr/weak_ptr）管理资源生命周期
- **场景**:
  - 动态内存自动释放
  - 避免内存泄漏
  - 循环引用检测（weak_ptr）
- **验收标准**: 能说出三种指针的适用场景，理解引用计数机制

#### C++11-Move-Semantics
- **描述**: 移动构造/移动赋值 + 右值引用 + 完美转发
- **场景**:
  - 避免不必要的拷贝
  - 资源转移而非复制
  - std::move 和 std::forward
- **验收标准**: 能用移动语义优化性能，理解移动 vs 拷贝的边界

#### C++11-Lambda
- **描述**: Lambda 表达式 + 捕获列表 + 高阶函数
- **场景**:
  - STL 算法配合
  - 即时函数对象
  - 闭包捕获状态
- **验收标准**: 能用 lambda 替代函数对象，理解捕获语义

#### C++11-Range-For
- **描述**: 范围 for 循环语法
- **场景**:
  - 遍历容器
  - 自动推导迭代器
- **验收标准**: 能用范围 for 简化循环代码

### C++14 特性

#### C++14-Auto
- **描述**: auto 类型推导深化
- **场景**:
  - 复杂类型简化
  - 模板元编程辅助
- **验收标准**: 理解 auto 推导规则和限制

### C++17 特性

#### C++17-Template
- **描述**: 模板进阶（类模板参数推导、变量模板）
- **场景**:
  - 模板特化与偏特化
  - 可变参数模板
- **验收标准**: 能编写泛型代码，理解 SFINAE 原则

#### C++17-Structured-Binding
- **描述**: 结构化绑定 `auto [a, b] = ...`
- **场景**:
  - pair/tuple 解包
  - struct 成员访问
- **验收标准**: 能用结构化绑定简化代码

#### C++17-TMP
- **描述**: 模板元编程（TypeList、编译期计算）
- **场景**:
  - 类型计算
  - constexpr if
- **验收标准**: 理解编译期计算的基本模式

#### C++17-SFINAE
- **描述**: SFINAE 原则 + enable_if + void_t
- **场景**:
  - 条件编译
  - 类型约束
- **验收标准**: 能用 SFINAE 实现类型过滤

### C++20 特性

#### C++20-Concept
- **描述**: 概念（Concepts）定义类型约束
- **场景**:
  - 清晰的接口约束
  - 友好的编译错误
- **验收标准**: 理解 Concepts vs SFINAE 的优势

#### C++20-Coroutines
- **描述**: 协程（co_await/co_yield/co_return）
- **场景**:
  - 异步编程
  - 生成器模式
- **验收标准**: 理解协程状态机和工作原理

#### C++20-Modules
- **描述**: 模块（Module）替代头文件
- **场景**:
  - 编译加速
  - 更好的封装
- **验收标准**: 理解模块化设计思想
