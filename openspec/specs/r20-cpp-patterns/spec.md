# R20 C++ 设计模式与工程实践能力规格

## 概述

R20 C++ 设计模式与工程实践能力涵盖 GoF 设计模式、SOLID 原则、现代 C++ 最佳实践，为高质量 C++ 代码设计奠定基础。

## 能力清单

### 创建型模式

#### R20-Creational
- **描述**: 单例、工厂、抽象工厂、建造者、原型模式
- **场景**:
  - 对象创建封装
  - 依赖注入
  - 配置管理
- **验收标准**: 能识别并正确应用创建型模式

### 结构型模式

#### R20-Structural
- **描述**: 适配器、桥接、组合、装饰器、外观、代理模式
- **场景**:
  - 接口适配
  - 对象组合
  - 装饰增强
- **验收标准**: 能用结构型模式解耦系统

### 行为型模式

#### R20-Behavioral
- **描述**: 策略、观察者、命令、迭代器、模板方法、访问者模式
- **场景**:
  - 算法封装
  - 事件驱动
  - 行为解耦
- **验收标准**: 能用行为型模式实现灵活交互

### SOLID 原则

#### R20-SOLID
- **描述**: SRP/OCP/LSP/ISP/DIP 五大原则
- **场景**:
  - 单一职责
  - 开闭原则
  - 里氏替换
  - 接口隔离
  - 依赖倒置
- **验收标准**: 能用 SOLID 原则审视和改进代码

### CRTP 模式

#### R20-CRTP
- **描述**: 奇特的递归模板模式 + 静态多态
- **场景**:
  - 取消虚函数开销
  - 编译期多态
  - 静态分发
- **验收标准**: 理解 CRTP 原理和适用场景

### Policy 模式

#### R20-Policy-Based
- **描述**: Policy 类模板 + 组合策略 + 编译期多态
- **场景**:
  - 库设计
  - 策略组合
  - 编译期配置
- **验收标准**: 能用 Policy 模式设计灵活库

### 类型擦除

#### R20-Type-Erasure
- **描述**: std::function + std::any + 类型擦除原理
- **场景**:
  - 异构容器
  - 回调封装
  - 泛型接口
- **验收标准**: 理解类型擦除机制和权衡

### Pimpl 惯用法

#### R20-Pimpl
- **描述**: 指针实现细节 + 编译防火墙 + ABI 稳定
- **场景**:
  - 头文件保护
  - 二进制兼容
  - 编译加速
- **验收标准**: 能用 Pimpl 惯用法改善头文件设计

### 表达式模板

#### R20-Expr-Template
- **描述**: 惰性求值 + 矩阵运算 + 表达式树
- **场景**:
  - 数值计算
  - 延迟求值
  - 性能优化
- **验收标准**: 理解表达式模板的惰性求值机制

### 代码质量工具

#### R20-Clang-Tidy
- **描述**: modernize-use-* + performance-* 检查
- **场景**:
  - 代码规范检查
  - 性能优化建议
  - 现代 C++ 迁移
- **验收标准**: 能用 clang-tidy 检查和改进代码
