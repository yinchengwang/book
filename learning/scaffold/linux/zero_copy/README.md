# 零拷贝技术

## 概述

本卡演示 Linux 零拷贝技术，包括 sendfile、mmap 等高性能 I/O 方式。

## 传统 I/O 流程

```
read/write 模式（4 次拷贝）：

  磁盘 ──DMA──> 内核缓冲区 ──CPU──> 用户缓冲区
                                         │
  网卡 <──DMA── socket缓冲区 <──CPU── 用户缓冲区
```

| 拷贝 | 操作 | 涉及 |
|------|------|------|
| 1 | 磁盘 → 内核缓冲区 | DMA |
| 2 | 内核缓冲区 → 用户缓冲区 | CPU |
| 3 | 用户缓冲区 → socket 缓冲区 | CPU |
| 4 | socket 缓冲区 → 网卡 | DMA |

## mmap 内存映射

```
mmap 模式（3 次拷贝）：

  磁盘 ──DMA──> 内核缓冲区 ──DMA──> 网卡
                       │
                       └──映射──> 用户空间
```

减少了一次用户态和内核态之间的拷贝。

## sendfile 零拷贝

```
sendfile 模式（2 次拷贝）：

  磁盘 ──DMA──> 内核缓冲区 ──DMA──> 网卡
              【完全在内核空间完成】
```

真正的零拷贝，数据无需进入用户态。

## API 总结

| 函数 | 说明 | 拷贝次数 |
|------|------|---------|
| `read/write` | 传统同步 I/O | 4 |
| `mmap/write` | 内存映射 | 3 |
| `sendfile` | 零拷贝 | 2 |

## 适用场景

| 场景 | 推荐方案 |
|------|---------|
| HTTP 文件下载 | sendfile |
| 日志处理 | mmap |
| 数据库存储 | read/write |
| 大文件传输 | sendfile |

## 编译运行

```bash
make              # 编译
make run          # 运行性能测试
make run-compare  # 仅运行对比
```

## 学习要点

1. **DMA** 直接内存访问，不占用 CPU
2. **用户态/内核态切换** 有性能开销
3. **sendfile** 适合大文件传输
4. **mmap** 适合需要处理数据的场景
