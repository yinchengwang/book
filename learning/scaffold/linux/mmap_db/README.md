# mmap_db — mmap 内存映射存储引擎

## 简介

演示 Linux mmap 内存映射机制在存储引擎中的应用，包括顺序/随机访问性能对比和预读策略。

## 编译

```bash
gcc -std=c11 -Wall -o mmap_db main.c
```

## 运行

```bash
./mmap_db
```

## 核心概念

### mmap 原理

```
进程虚拟地址空间         内核                 磁盘
┌──────────────┐     ┌──────────────┐    ┌──────────┐
│ mapped_addr  │────→│ 页表        │───→│ 文件数据 │
│ (直接访问)    │     │ (按需换入)   │    └──────────┘
└──────────────┘     └──────────────┘
```

### 与传统 I/O 对比

- 传统 read/write：内核缓冲区 → 用户缓冲区（2次拷贝）
- mmap：直接映射，无拷贝

## 关键 API

- `mmap()`：创建映射
- `munmap()`：解除映射
- `msync()`：同步到磁盘
- `madvise()`：提示预读策略
