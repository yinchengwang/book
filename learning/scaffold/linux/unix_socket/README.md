# Unix Domain Socket

## 概述

Unix Domain Socket（Unix 域套接字）是一种本地进程间通信机制，使用与 TCP/UDP 相同的套接字 API，但通过文件系统路径而非网络地址进行寻址。它是最快的 IPC（Inter-Process Communication）方式之一。

## 与 TCP/UDP 对比

| 特性 | Unix Domain Socket | TCP/UDP (127.0.0.1) |
|------|-------------------|---------------------|
| 寻址方式 | 文件系统路径（如 `/tmp/sock`） | IP 地址 + 端口号 |
| 数据传输 | 同一主机内核内部 | 经过网络协议栈 |
| 性能 | 最高（零拷贝优化） | 较低（协议开销） |
| 安全性 | 通过文件系统权限控制 | 依赖防火墙/端口 |
| 广播/多播 | 不支持 | 支持 |

## stream vs dgram 模式

### SOCK_STREAM（流式套接字）
- **面向连接**：类似 TCP，需要先建立连接
- **可靠传输**：保证数据顺序和不丢失
- **字节流**：无消息边界，可能发生粘包
- **适用场景**：需要可靠传输的持续会话

### SOCK_DGRAM（数据报套接字）
- **无连接**：无需建立连接，直接发送
- **独立消息**：保留消息边界
- **有长度限制**：通常 2KB 左右
- **适用场景**：简短独立的消息交换

## 编译与运行

```bash
# 编译
make

# 运行
make run

# 清理
make clean
```
