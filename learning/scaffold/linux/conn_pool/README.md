# Linux 连接池学习卡片

本学习卡片演示连接池的实现原理，包括预创建连接、获取/归还机制、超时回收、以及并发访问控制。

## 学习目标

1. 理解连接池的核心概念
2. 掌握连接池的基本实现
3. 理解并发访问控制（互斥锁）
4. 理解空闲连接回收机制

## 核心概念

### 连接池架构

```
应用程序                    连接池                    数据库
    |                         |                        |
    |-- get_conn()  -------->|                        |
    |                         |-- socket() + connect()-->|
    |<-- conn  ---------------|                        |
    |                         |                        |
    | 使用连接...             |                        |
    |                         |                        |
    |-- return_conn() ------>|                        |
    |                         |<-- close() ------------|
    |                         |                        |
```

### 核心操作

| 操作 | 说明 | 线程安全 |
|------|------|----------|
| pool_init() | 初始化池，预创建连接 | 是 |
| get_conn() | 获取可用连接 | 是 |
| return_conn() | 归还连接到池 | 是 |
| recycle_idle() | 回收空闲连接 | 是 |
| pool_destroy() | 销毁连接池 | 是 |

### 连接状态

```
空闲 -> 使用中 -> 归还 -> 空闲
  |                   |
  +---- 空闲超时 ----->+
         (回收)
```


```bash
make clean
```
