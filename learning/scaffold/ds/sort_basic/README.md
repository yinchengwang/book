# 数据结构学习卡片：基础排序 (Sort - Basic)

## 学习目标

1. 理解排序算法的稳定性概念及其重要性
2. 掌握冒泡排序、选择排序、插入排序的原理和实现
3. 理解三种排序算法的时间复杂度分析
4. 掌握 in-place（原地）和 out-of-place（非原地）的区别
5. 理解稳定排序和不稳定排序的应用场景

## 核心概念

### 排序基础

- **排序**：将一组数据按照关键字进行有序排列
- **原地排序**：不需要额外存储空间，O(1) 空间复杂度
- **非原地排序**：需要额外的存储空间，O(n) 或更高空间复杂度

### 稳定性

- **定义**：排序后相同关键字元素的相对顺序保持不变
- **稳定排序**：冒泡排序、插入排序、归并排序
- **不稳定排序**：选择排序、快速排序、堆排序
- **重要性**：
  - 多字段排序：先按次要字段排序，再按主要字段稳定排序
  - 保持业务逻辑：如考试成绩相同按学号保持原有顺序

### 时间复杂度

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 冒泡排序 | O(n) | O(n^2) | O(n^2) | O(1) | 是 |
| 选择排序 | O(n^2) | O(n^2) | O(n^2) | O(1) | 否 |
| 插入排序 | O(n) | O(n^2) | O(n^2) | O(1) | 是 |

### 冒泡排序

- **核心思想**：相邻元素两两比较，大的往后放
- **优化**：提前终止（当某趟没有交换时说明已有序）
- **特点**：
  - 稳定
  - 简单但效率低
  - 适合小规模或近乎有序数据

### 选择排序

- **核心思想**：从未排序部分选择最小/最大元素，与未排序部分第一个交换
- **特点**：
  - 不稳定（相同值的位置可能改变）
  - 交换次数固定 O(n)
  - 时间复杂度恒定 O(n^2)

### 插入排序

- **核心思想**：将元素插入到已排序部分的正确位置
- **特点**：
  - 稳定
  - 对于近乎有序的数据效率很高
  - 是其他高级排序的基础（如 Shell 排序）

### in-place vs out-of-place

- **原地排序 (in-place)**：
  - 冒泡、选择、插入、堆排序、快速排序
  - 只需要 O(1) 额外空间

- **非原地排序 (out-of-place)**：
  - 归并排序
  - 需要 O(n) 额外空间用于合并
