# 数据结构学习卡片：队列 (Queue)

## 学习目标

1. 理解队列的 FIFO（先进先出）特性
2. 掌握循环队列的实现原理（数组 + 取模）
3. 掌握链式队列的实现原理（链表 + 双指针）
4. 理解阻塞队列的概念与应用场景
5. 理解队列与双端队列（Deque）的区别

## 核心概念

### 队列基础

- **定义**：一种特殊的线性表，只允许在一端插入（队尾），另一端删除（队头）
- **特性**：FIFO（First In First Out）
- **基本操作**：
  - `enqueue` / `push`：入队，从队尾插入
  - `dequeue` / `pop`：出队，从队头删除
  - `front`：获取队头元素
  - `rear`：获取队尾元素

### 循环队列

- **背景**：普通数组队列出队后无法复用空出的空间
- **原理**：通过取模运算实现逻辑上的"环"
- **关键点**：
  - `head`：指向队头元素
  - `tail`：指向下一个入队位置
  - `count`：记录元素个数
  - 满/空判断：`count == capacity` / `count == 0`
- **优点**：空间利用率高 O(1) 入队出队
- **缺点**：需要预先确定容量

### 链式队列

- **原理**：用链表实现，队头在链表头部，队尾在链表尾部
- **关键点**：
  - `front`：指向链表头节点
  - `rear`：指向链表尾节点
  - 入队：在尾部插入节点
  - 出队：从头部删除节点
- **优点**：空间按需分配，无容量限制
- **缺点**：每个节点有额外指针开销

### 阻塞队列

- **定义**：一种线程安全的队列，支持阻塞操作
- **特性**：
  - 队列为空时，`dequeue` 阻塞等待
  - 队列满时，`enqueue` 阻塞等待
- **应用**：
  - 生产者-消费者模式
  - 线程池任务队列
  - 消息队列

### 队列 vs 双端队列

| 特性 | 队列 (Queue) | 双端队列 (Deque) |
|------|-------------|-----------------|
| 队头插入 | 不支持 | 支持 |
| 队头删除 | 支持 | 支持 |
| 队尾插入 | 支持 | 支持 |
| 队尾删除 | 不支持 | 支持 |
| 应用场景 | 任务调度、缓冲 | 滑动窗口、撤销栈 |

### 时间复杂度

| 操作 | 循环队列 | 链式队列 |
|------|---------|---------|
| 入队 `enqueue` | O(1) | O(1) |
| 出队 `dequeue` | O(1) | O(1) |
| 获取队头 `front` | O(1) | O(1) |
| 获取队尾 `rear` | O(1) | O(1) |
