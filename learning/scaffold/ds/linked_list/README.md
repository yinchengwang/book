# 数据结构学习卡片：链表 (Linked List)

## 学习目标

1. 理解链表的节点结构和指针概念
2. 掌握单链表的创建、插入、删除、翻转操作
3. 掌握双链表的结构特点和 O(1) 操作优势
4. 理解链表与数组的性能差异和适用场景
5. 理解哨兵节点的作用

## 核心概念

### 链表基础

- **定义**：由节点组成的线性数据结构，每个节点包含数据和指向下一个节点的指针
- **节点结构**：
  - `data`：数据域，存储数据
  - `next`（单链表）/ `prev + next`（双链表）：指针域
- **特点**：物理存储不连续，通过指针连接

### 单链表

- **结构**：每个节点只有 `next` 指针，只能从头向尾遍历
- **哨兵节点**：头部添加一个哨兵节点，简化边界处理
- **操作复杂度**：
  - 头部插入/删除：O(1)
  - 尾部插入：O(n)（需遍历）
  - 按值查找/删除：O(n)

### 双链表

- **结构**：每个节点有 `prev` 和 `next` 指针，支持双向遍历
- **双向哨兵**：头尾都添加哨兵节点，简化边界处理
- **操作复杂度**：
  - 头部/尾部插入/删除：O(1)
  - 按值查找/删除：O(n)
- **空间代价**：每个节点多一个指针（8 字节 64 位系统）

### 链表翻转（迭代）

```
原链表: 1 -> 2 -> 3 -> NULL
翻转过程:
1. prev=NULL, curr=1
2. curr->next=NULL, prev=1, curr=2
3. curr->next=1, prev=2, curr=3
4. curr->next=2, prev=3, curr=NULL
结果: 3 -> 2 -> 1 -> NULL
```

### 哨兵节点的作用

- **简化边界判断**：不需要检查 `NULL` 指针
- **统一操作**：头插、头删操作与普通节点相同
- **示例**：
  - 无哨兵：需要单独处理空链表、第一个节点
  - 有哨兵：所有操作统一处理

### 时间复杂度对比

| 操作 | 数组 | 单链表 | 双链表 |
|------|------|--------|--------|
| 随机访问 | O(1) | O(n) | O(n) |
| 头部插入 | O(n) | O(1) | O(1) |
| 头部删除 | O(n) | O(1) | O(1) |
| 尾部插入 | O(1) | O(n) | O(1) |
| 尾部删除 | O(1) | O(n) | O(1) |
| 中间插入 | O(n) | O(n) | O(n) |
| 按值删除 | O(n) | O(n) | O(n) |
