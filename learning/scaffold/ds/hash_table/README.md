# 数据结构学习卡片：哈希表 (Hash Table)

## 学习目标

1. 理解哈希函数的作用与常见算法（djb2、FNV-1a）
2. 掌握哈希冲突的解决方法：开放地址法（线性探测）与链地址法
3. 理解负载因子的概念及其对性能的影响
4. 掌握哈希表的动态扩容（rehashing）机制

## 核心概念

### 哈希函数

- **作用**：将任意长度的键转换为固定范围的桶索引
- **理想特性**：均匀分布、确定性、高效计算
- **常用算法**：
  - djb2：`hash = hash * 33 + c`
  - FNV-1a：`hash = hash XOR c * FNV_prime`
  - MurmurHash：高性能非加密哈希

### 冲突解决策略

| 策略 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 线性探测 | 冲突后顺序查找下一个空位 | 简单、缓存友好 | 容易产生聚集 |
| 二次探测 | 冲突后用二次方步长探测 | 减少聚集 | 探测序列可能不均匀 |
| 链地址法 | 每个桶存储链表 | 插入方便 | 额外内存开销 |
| 开放地址法 | 在表中寻找空位 | 无额外指针 | 删除复杂（需墓碑） |

### 负载因子

- **定义**：`α = n / m`（n=元素数，m=桶数）
- **影响**：负载因子越高，冲突越频繁，查找越慢
- **阈值**：通常 0.7~0.75 时触发扩容
- **时间复杂度**：
  - 理想情况：O(1)
  - 最坏情况：O(n)

### 扩容与再哈希

- **扩容时机**：负载因子超过阈值时
- **扩容策略**：通常翻倍（2x）
- **再哈希（rehashing）**：扩容后重新计算所有元素位置
- **均摊复杂度**：追加操作仍是 amortized O(1)

### 时间复杂度

| 操作 | 平均 | 最坏 |
|------|------|------|
| 插入 | O(1) | O(n) |
| 查找 | O(1) | O(n) |
| 删除 | O(1) | O(n) |
