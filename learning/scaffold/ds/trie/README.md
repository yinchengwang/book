# Trie（字典树）学习卡片

## 学习目标

1. 理解 Trie 的基本结构：根节点、字符路径、终止标记
2. 掌握 Trie 的核心操作：插入、查找、前缀匹配
3. 了解压缩字典树（Radix Tree）的概念与优化
4. 理解 Trie 在实际工程中的应用场景

## 核心概念

### Trie 树结构

- **节点**：每个节点代表一个字符位置
- **边**：字符连接，表示从前缀到下一个字符的转移
- **终止标记**：标记以该节点结尾的完整单词
- **根节点**：空字符串的前缀

### 时间复杂度

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 插入 | O(m) | m = 单词长度 |
| 查找 | O(m) | 精确匹配 |
| 前缀匹配 | O(m) | m = 前缀长度 |
| 空间 | O(ALPHABET_SIZE * m * n) | 与字符集和单词数相关 |

### 与其他数据结构对比

| 数据结构 | 查找方式 | 适用场景 |
|----------|----------|----------|
| 哈希表 | 键值映射 | 精确查找，无前缀需求 |
| Trie | 前缀树 | 前缀搜索、字典补全 |
| B-Tree | 排序树 | 范围查询、数据库索引 |
| Radix Tree | 压缩前缀树 | 大规模字符串集合 |
