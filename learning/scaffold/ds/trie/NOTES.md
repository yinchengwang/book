# Trie 工程对照笔记

## 前缀搜索的工程应用

### 1. IP 路由与最长前缀匹配（LPM）

在计算机网络中，路由表需要根据目的 IP 地址找到最匹配的路由条目。传统的精确匹配无法满足需求，因为路由条目是 IP 前缀（如 `192.168.0.0/16`），需要找到最长匹配的前缀。

**Trie 的应用**：
- 将 IP 地址转换为二进制 Trie（或十六进制 Trie）
- 每个节点代表一个网络位
- 查找时从根到叶，找到终止标记且路径最长的条目
- Linux 内核使用 Radix Tree（压缩 Trie）管理路由缓存

**工程案例**：
- Linux 内核 `fib_trie`：IP 路由查找
- Cisco 路由器：TCAM 与 Trie 结合的硬件加速
- DPDK：高性能网络数据包处理中的前缀树

### 2. 搜索引擎自动补全

当用户在搜索框输入部分文字时，搜索引擎需要快速返回可能的完整词条。

**Trie 的应用**：
- 将词库预先构建为 Trie
- 根据用户输入的前缀快速定位到 Trie 中的节点
- 递归收集该节点下的所有完整单词
- 按热度/频率排序返回

**工程案例**：
- Google 搜索建议：实时补全
- IDE 代码补全：变量名、函数名快速检索
- 命令行工具：`bash-completion`、`fish` 自动补全

### 3. 词典与拼写检查

文字处理软件需要验证单词拼写，并提供纠正建议。

**Trie 的应用**：
- 词库构建 Trie
- 输入单词前缀，检查是否存在
- 收集相近前缀的单词作为建议
- 结合编辑距离算法（如 Levenshtein）提供纠错

**工程案例**：
- Hunspell：开源拼写检查库
- Chrome 浏览器：地址栏 URL 补全
- Microsoft Word：拼写错误下划线标记

### 4. 字符串集合的高效存储

在内存受限场景（如嵌入式系统），需要高效存储大量字符串。

**Radix Tree 优化**：
- 合并连续的单字符边，减少节点数
- 每条边可以存储多个字符
- 典型应用：Linux 内核 `radix_tree` 用于页面缓存管理

**工程案例**：
- Linux 内核：页面缓存索引、inode 管理
- Nginx：配置项的 radix tree 索引
- Redis：有序集合的紧凑编码

### 5. 字符串排序与去重

在大数据处理中，需要对海量字符串进行排序。

**Trie 的优势**：
- 天然有序：深度优先遍历即得到字典序
- 去重简单：只需检查终止标记
- 无需额外排序步骤

**工程案例**：
- 分布式排序：MapReduce 中的 partition
- 数据库索引：字符串类型列的索引结构

### 6. 文本压缩

基于 Trie 的算法（如 LZW）可以用于文本压缩。

**原理**：
- 扫描文本，逐步构建 Trie
- 遇到未知模式时输出编码
- 将新模式加入 Trie

**工程案例**：
- Unix `compress` 命令
- PDF 内嵌字体编码

## 工程实现要点

### 内存优化

1. **子节点存储**：
   - 固定数组：`ALPHABET_SIZE * sizeof(pointer)`，简单但浪费内存
   - 动态数组：按需分配，复杂度增加
   - 压缩存储：字典压缩或伙伴系统

2. **字符集处理**：
   - ASCII（26 字母）：简单数组
   - 扩展字符集：使用 `std::unordered_map` 或 hash
   - 二进制数据：使用 2叉 Trie（0/1 边）

3. **节点结构精简**：
   ```c
   typedef struct {
       struct TrieNode *next[ALPHABET_SIZE];
       bool is_end;
   } TrieNode;
   ```

### 性能优化

1. **批量构建**：词库较大时使用批量插入，避免频繁分配
2. **内存池**：预分配节点池，减少 `malloc` 调用
3. **延迟删除**：标记删除而非立即释放，保留 Trie 结构
4. **缓存友好**：将子节点数组放在结构体开头，提高缓存命中率

### 并发安全

1. **读写锁**：读操作可并发，写操作需独占
2. **无锁 Trie**：使用 CAS（Compare-And-Swap）实现
3. **分片 Trie**：将 Trie 按前缀分片，并发写入不同分片

## 总结

Trie 是处理字符串前缀搜索的利器，在网络路由、搜索补全、拼写检查等领域广泛应用。其核心价值在于：

- **前缀高效**：O(m) 时间复杂度找到所有以某前缀开头的字符串
- **天然有序**：遍历即得到字典序
- **可扩展**：支持词频统计、最长公共前缀等多种扩展

在实际工程中，需要根据数据规模和性能要求选择合适的 Trie 变体（普通 Trie、Radix Tree、双数组 Trie 等）。
