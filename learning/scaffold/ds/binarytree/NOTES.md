# 工程对照说明

## 教学代码 vs 工程实现

### 数据结构对比

| 方面 | 教学代码 (learning/scaffold) | 工程实现 (engineering/src/db) |
|------|------------------------------|-------------------------------|
| 树类型 | 二叉树 (Binary Tree) | B-Tree、B+Tree、R-Tree 等 |
| 节点定义 | 简单 struct，包含左右指针 | 复杂结构体，包含 records 数组 |
| 比较方式 | 整数直接比较 | 可插拔 compare 函数 |
| 存储方式 | 内存动态分配 | 磁盘持久化支持 |

### 工程中的树结构实现

`engineering/src/db/index/` 下实现了多种树索引结构：

1. **BTree (`btree/`)**
   - 最小度 t，每个节点最多 2t-1 个 key
   - 内部节点和叶子节点都保存 key-value
   - `btree_private.h` 中定义节点结构：
     ```c
     typedef struct btree_node {
         bool is_leaf;
         uint32_t nkeys;
         btree_record_t *records;
         struct btree_node **children;
     } btree_node_t;
     ```

2. **B+Tree (`bplus_tree/`)**
   - 仅叶子节点保存数据，内部节点仅作索引
   - 叶子节点通过链表连接，支持顺序扫描
   - 优化磁盘访问模式

3. **R-Tree (`rtree/`)**
   - 用于空间数据索引（矩形/多边形）
   - 支持范围查询和最近邻搜索

4. **Radix Tree (`radix_tree/`)**
   - 前缀树变体，用于字符串键索引
   - PostgreSQL 中用于缓存键管理

### 核心差异

| 特性 | 教学二叉树 | 工程 B-Tree |
|------|----------|-------------|
| 用途 | 算法学习 | 数据库索引 |
| 持久化 | 内存 | 磁盘 |
| 多路 | 二叉（2路） | 多路（2t路） |
| 平衡 | 递归平衡 | 分裂/合并操作 |
| 比较 | 固定整数比较 | 可扩展比较器 |

### 工程启示

1. **compare 函数设计**：工程中所有树结构都支持自定义比较器，便于扩展
2. **节点抽象**：`btree_node_t` 将数据和元数据分离，便于统一管理
3. **持久化考虑**：工程代码考虑磁盘页对齐，而教学代码仅关注算法逻辑
4. **内存管理**：工程代码使用更复杂的内存池，教学代码使用简单的 malloc/free

### 学习路径

1. 先掌握教学代码的遍历、插入、删除算法
2. 理解递归在树操作中的应用
3. 再学习工程代码中的 B-Tree 分裂/合并逻辑
4. 最终理解为何数据库选择 B-Tree 作为主索引结构
