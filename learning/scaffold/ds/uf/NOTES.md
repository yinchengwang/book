# 工程对照说明

## 与工程代码的对应关系

学习轨（learning/）的 scaffold 侧重于**算法原理与直观理解**，而工程轨（engineering/）则关注**生产级实现与工程约束**。本 NOTES 对照说明并查集在工程中的实际应用。

---

## 1. 连通性判断在工程中的应用

### 1.1 数据库死锁检测

在数据库存储引擎中，并查集用于**死锁检测**。当事务 T1 持有锁 A 等待锁 B，事务 T2 持有锁 B 等待锁 A 时，形成环路。可用并查集将等待图中的节点分组：若新产生的等待关系使两个已有关键字集合变为连通，则检测到死锁，此时需要选择回滚代价最小的事务进行中止。

### 1.2 分布式系统节点分组

在分布式数据库（如 Cassandra、TiDB）中，节点通过网络组成集群。并查集用于**拓扑感知路由**：当网络发生分区（Network Partition）时，节点通过心跳相互探测，连通分量变化的节点被标记为不可用，数据复制只在同一连通分量内进行，保证一致性边界清晰。

### 1.3 编译器等价语句归类

在数据库查询优化器中，**等价类推导**使用并查集。例如 `WHERE a = b AND b = c` 可推导出 `a`、`b`、`c` 属于同一等价类。并查集将所有通过等值条件关联的列归为一组，后续优化阶段可利用等价性进行谓词下推、索引利用等改写。

### 1.4 容器网络微分段

在 Kubernetes 网络策略或云安全组中，**微分段（Microsegmentation）** 依赖并查集追踪服务间可达性。当 Pod 或 VM 的标签发生变化时（加入/离开 Service Mesh），并查集快速判定网络策略是否允许新的通信路径，避免遍历整个网络拓扑。

### 1.5 游戏服务器组队匹配

多人游戏中，**组队逻辑**常用并查集管理已组队玩家。玩家发起组队请求时，union 操作将两个队伍合并；退出队伍时需要分裂集合（标准并查集不支持，但可用并查集的变体或重新构建）。队伍内的玩家可共享游戏状态同步。

## 2. 本实现与工程实现的差异

| 方面 | Scaffold 实现 | 工程实现 |
|------|-------------|---------|
| 存储 | 数组（简单直观） | 可能用链表 + 跳跃表提升分裂效率 |
| 并发 | 单线程 | 多线程加锁或无锁 CAS（参考 PostgreSQL SSI） |
| 分裂支持 | 无（仅支持合并） | 图结构或持久化日志支持分裂 |
| 持久化 | 内存 | WAL 写前日志 + Checkpoint 恢复 |
