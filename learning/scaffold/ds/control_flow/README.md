# 数据结构学习卡片：控制流 (Control Flow)

## 学习目标

1. 掌握条件语句 if-else、switch 的使用场景
2. 理解三种循环结构 for、while、do-while 的区别
3. 理解有限状态自动机（FSM）模式
4. 掌握 goto 的合法使用场景（错误处理清理）

## 核心概念

### 条件语句

#### if-else 语句

- **使用场景**：范围判断、复合条件
- **else-if 链**：多条件分支，比嵌套 if 更清晰
- **分支预测**：现代 CPU 会预测分支走向，预测失败有性能损失

#### switch 语句

- **使用场景**：离散值的多分支判断
- **优点**：编译器可优化为跳转表，O(1) 复杂度
- **注意事项**：每个 case 需显式 break，否则会贯穿执行

### 循环结构

#### for 循环

- **适用场景**：已知迭代次数
- **语法**：`for (初始化; 条件; 更新)`
- **循环变量**：建议使用局部变量，避免污染外层作用域

#### while 循环

- **适用场景**：未知迭代次数，前置条件判断
- **特点**：条件不满足时可能一次都不执行

#### do-while 循环

- **适用场景**：至少执行一次的场景
- **特点**：后置条件判断，循环体至少执行一次

### 状态机模式

- **定义**：有限状态自动机（Finite State Machine, FSM）
- **组成**：状态集、转换函数、初始状态、接受状态
- **应用**：协议解析（HTTP、TCP）、词法分析、游戏 AI
- **优点**：逻辑清晰、易于测试、可视化友好

### goto 的合法使用

- **适用场景**：多层嵌套函数退出时的统一清理
- **优点**：避免重复的清理代码，保持错误处理逻辑集中
- **不推荐**：跨函数跳转、正常业务逻辑中滥用
- **最佳实践**：goto 只用于错误处理，统一跳转到 cleanup 标签
