# 数据结构学习卡片：动态内存 (Dynamic Memory)

## 学习目标

1. 理解堆内存与栈内存的区别
2. 掌握 malloc/calloc/realloc/free 的用法和适用场景
3. 识别并避免常见的内存错误
4. 理解内存分配失败的处理方法

## 核心概念

### 内存布局

```
高地址 ────────────────────────
    内核空间
───────────────────────────────
    栈 (Stack)
    - 向低地址增长
    - 自动管理，函数返回时释放
    - 局部变量、函数参数
───────────────────────────────
    ↓ 增长方向
    ↑
───────────────────────────────
    堆 (Heap)
    - 向高地址增长
    - 手动管理 (malloc/free)
    - 动态分配的数据
───────────────────────────────
    BSS 段 (未初始化全局/静态)
───────────────────────────────
    数据段 (已初始化全局/静态)
───────────────────────────────
    代码段 (Text)
低地址 ────────────────────────
```

### 分配函数对比

| 函数 | 分配方式 | 初始化 | 适用场景 |
|------|---------|--------|---------|
| `malloc` | 分配指定字节数 | 不初始化 | 需要手动初始化、已知字节大小 |
| `calloc` | 分配 n * size 字节 | 初始化为 0 | 数组、需要零初始化的场景 |
| `realloc` | 调整已分配内存 | 新空间不初始化 | 动态扩展/缩减数组 |

### 常见内存错误

#### Double Free

- **错误**：对同一块内存调用两次 free
- **后果**：堆元数据损坏，可能被利用进行攻击
- **预防**：释放后将指针设为 NULL，使用智能指针（C++）

#### Use After Free

- **错误**：访问已释放的内存
- **后果**：读到垃圾值、段错误、或被攻击者控制
- **预防**：释放后将指针设为 NULL，缩短生命周期

#### Memory Leak

- **错误**：分配后忘记释放
- **后果**：内存逐渐耗尽，系统变慢
- **预防**：遵循分配/释放配对原则，使用 RAII（C++）

#### Buffer Overflow

- **错误**：写入超过分配大小的数据
- **后果**：覆盖相邻内存，可能被利用
- **预防**：使用安全函数、边界检查
