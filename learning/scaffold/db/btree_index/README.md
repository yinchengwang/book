# btree_index - BTree 索引原理

## 概述

BTree（B+树）是一种自平衡的多路搜索树，广泛应用于数据库索引实现。

## 节点类型

### Internal（内部节点）
- 存储键和子页面指针
- 键用于分割子树
- 不存储实际数据

### Leaf（叶子节点）
- 存储键和 TID（Tuple ID）
- TID 指向堆表中的实际记录
- 叶子节点之间通过双向链表连接

## 页面结构

```
+-------------------+
| PageHeader        |  24 bytes
| - type            |
| - level           |
| - num_keys        |
| - free_offset     |
+-------------------+
| Entries[]         |  可变长度
| - key             |
| - child/TID       |
+-------------------+
| Free Space        |
+-------------------+
```

## 核心操作

### 搜索
从根节点开始，根据键值逐层向下查找。

### 插入
1. 定位插入位置
2. 如果叶子节点未满，直接插入
3. 如果已满，执行页面分裂

### 分裂策略
当叶子节点达到 `MAX_KEYS` 时，分裂为两个节点：
- 左节点保留前 `MAX_KEYS/2` 个条目
- 右节点存储后 `MAX_KEYS/2` 个条目
- 分裂键上推至父节点

### 范围扫描
1. 定位起始位置
2. 沿叶子节点链表顺序扫描
3. 直到超出范围边界
