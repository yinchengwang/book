# 数据库锁机制学习卡

## 概述

本卡片演示数据库并发控制中的锁机制，包括：
- 共享锁（S锁）和排他锁（X锁）
- 锁兼容性规则
- 锁升级（S -> X）
- 锁粒度（行锁、页锁、表锁）

## 编译与运行

```bash
# 编译
gcc -std=c11 -Wall -o locking main.c

# 运行
./locking
```

## 学习要点

### 1. 锁类型

| 锁类型 | 符号 | 用途 | 并发能力 |
|--------|------|------|----------|
| 共享锁 | S | 读取数据 | 多个事务可同时持有 |
| 排他锁 | X | 修改数据 | 独占，同一资源只能一个 |

### 2. 锁兼容性矩阵

| 已持有 \ 请求 | S锁 | X锁 |
|-------------|-----|-----|
| 无锁        | Y   | Y   |
| S锁         | Y   | N   |
| X锁         | N   | N   |

- Y = 兼容（可以授予）
- N = 不兼容（阻塞或拒绝）

### 3. 锁升级

事务持有 S锁 后，可升级为 X锁，但必须满足：
- 只有一个S锁持有者（自身）
- 不能有其他事务同时持有S锁

### 4. 锁粒度

```
粒度越小 → 并发度越高 → 开销越大
粒度越大 → 并发度越低 → 开销越小

行锁(row)  → 粒度最细
页锁(page) → 中间粒度
表锁(table) → 粒度最粗
```

## 测试场景

1. **共享锁并发读** - 多个事务同时读取同一资源
2. **写操作阻塞** - S锁存在时X锁被拒绝
3. **锁升级** - S锁升级为X锁的流程
4. **升级冲突** - 多持有者时无法升级
5. **锁粒度** - 不同粒度的锁定范围
