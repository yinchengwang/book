# 两阶段提交（2PC）学习卡

## 概述

本卡片演示分布式事务的两阶段提交（Two-Phase Commit）协议，通过模拟协调者和参与者展示：
- Prepare 阶段：询问所有参与者是否可以提交
- Commit 阶段：所有参与者准备好后执行提交
- 故障恢复：协调者和参与者的恢复流程

## 编译与运行

```bash
# 编译
gcc -std=c11 -Wall -o 2pc main.c

# 运行
./2pc
```

## 学习要点

1. **2PC 协议**：Prepare → Commit/Rollback 两阶段
2. **协调者角色**：收集 Prepare 响应，决定提交或回滚
3. **参与者角色**：接收指令，写入日志，释放资源锁
4. **故障恢复**：通过日志状态决定后续操作
5. **局限性**：协调者崩溃可能导致阻塞（三阶段提交解决）
