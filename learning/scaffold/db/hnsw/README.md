# hnsw - HNSW 向量索引原理

## 概述

HNSW（Hierarchical Navigable Small World，分层可导航小世界图）是一种基于图的近似最近邻搜索算法。通过构建多层邻居图，在搜索时从高层稀疏图快速下降到目标区域，再在底层密集图中做精细扩展。

**适用场景**：十万至百万级稠密向量，要求毫秒级低延迟、支持增量插入。

## 核心思想

```
         Level 2 (最稀疏)         ●────●        ← 入口点
                                  │
         Level 1                 ●────●────●
                                 │    │
         Level 0 (最密集)   ●────●────●────●────●  ← 所有点都在此层
```

- 上层图稀疏：节点少、连边少，搜索"跳得快"
- 底层图密集：所有节点都在第 0 层，搜索"搜得准"
- 节点层高由指数分布随机决定，越高层节点越少
- 搜索时从最高层入口点逐层贪心下降

## 核心算法

### 贪心搜索

从最高层入口点出发，在每层遍历邻居找到比当前更近的点，直到收敛，然后下降到下一层。

```
while true:
    记录当前最近点 prev
    遍历当前节点的邻居，计算距离
    找到比 prev 更近的点，更新 nearest
    if nearest == prev:  本层已收敛，break
```

### 插入算法

1. 随机采样层高（指数分布）
2. 从入口点贪心下降到 `pt_level + 1`
3. 在每层搜索 `ef_construction` 个最近邻居
4. 启发式选边（shrink）：避免邻居相互遮挡
5. 双向连接新节点与选中的邻居

### 扁平化邻接表

所有层级的邻接信息存储在单一扁平数组中，通过偏移量定位：

```
节点结构:
  neighbors[]     - 各层邻居（简化：共用数组）
  layer_offset    - 各层邻居的起始偏移
```

## 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `HNSW_M` | 4 | 每层最大邻居数（底层为 2*HNSW_M） |
| `ef_construction` | 16 | 构图搜索候选数，越大图质量越高 |
| `ef_search` | 10 | 搜索候选数，越大召回越高 |
