# mvcc — MVCC 多版本并发控制

## 核心概念

- **t_xmin/t_xmax**：事务边界标识
  - `t_xmin`：创建元组版本的事务ID
  - `t_xmax`：删除元组版本的事务ID（0表示未删除）

- **版本链**：链表组织多版本
  - 每个元组可能有多个版本，按时间顺序链接
  - 新版本指向旧版本，形成链式结构
  - 读取时遍历版本链，找到对当前快照可见的版本

- **快照**：active_txns 活跃事务列表
  - 记录事务开始时所有活跃的事务ID
  - 用于判断元组版本对当前事务是否可见

## 编译运行

```bash
make && ./test
```

## 核心算法

### 可见性判断规则

```
可见性 = (t_xmin 在快照中可见) AND (t_xmax == 0 OR t_xmax 在快照中不可见)
```

1. `t_xmin` 事务必须已提交或在快照中
2. 如果 `t_xmax > 0`，则删除事务必须未提交才可见
3. 否则该版本对当前事务不可见

### 版本链遍历

```
for each version in chain:
    if version_visible(version, snapshot):
        return version  // 返回最新可见版本
return NULL  // 无可见版本
```
