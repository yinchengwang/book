# python-generator 学习笔记

## 概念地图

Python 生成器是惰性求值的迭代器实现：

- **yield 语句**：函数包含 yield 即成为生成器函数，调用返回生成器对象
- **惰性求值**：按需产生值，不一次性加载全部数据到内存
- **生成器表达式**：类似列表推导式，但用圆括号，惰性求值
- **itertools**：标准库提供的函数式编程工具，操作迭代器

## 生成器 vs 列表

| 特性 | 生成器 | 列表 |
|------|--------|------|
| 内存 | O(1) | O(n) |
| 速度 | 首次访问稍慢 | 随机访问快 |
| 长度 | 可无限 | 有限 |
| 复用 | 一次性 | 可多次迭代 |

## 踩坑记录

1. **生成器只能迭代一次**：`list(gen)` 后再迭代返回空
2. **生成器不保留中间状态**：无法回退或重启
3. **yield from**：委托给子生成器，等价于手动循环 yield

## 工程对照（≥100 字硬约束）

惰性计算在 `learning/` 脚本中有实际应用：

1. **文件处理管道**：`sync-pipeline.py` 处理大量文件时，可用生成器逐行读取，避免内存爆炸
2. **itertools.chain**：`itertools.chain(*dirs)` 连接多个目录的文件列表
3. **内存优化**：大日志文件分析用 `for line in open('big.log')` 而非 `readlines()`
4. **生成器管道**：`filter -> map -> reduce` 模式处理数据流
5. **itertools.groupby**：对扫描结果按类型分组
6. **yield from**：在递归遍历目录时委托给子生成器

学完本卡能动手的事：将大文件处理脚本从列表推导式改为生成器表达式，减少内存占用。
