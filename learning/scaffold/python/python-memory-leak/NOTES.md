# python-memory-leak 学习笔记

## 概念地图

Python 内存泄漏场景及修复：

| 泄漏类型 | 原因 | 修复方案 |
|----------|------|----------|
| 全局列表 | 不断 append | 定期清理或用 deque |
| 闭包引用 | 闭包捕获大对象 | 避免引用不必要数据 |
| 缓存泄漏 | 无限增长 | LRU 缓存 |
| 循环引用 | 互相引用 | weakref 或手动清理 |

## 检测工具

1. **tracemalloc**：追踪内存分配
2. **objgraph**：可视化对象关系
3. **memory_profiler**：逐行内存分析
4. **gc.get_objects()**：查看所有对象

## 踩坑记录

1. **全局状态**：全局列表/字典容易被忽视
2. **回调泄漏**：事件监听器未移除
3. **缓存膨胀**：没有限制的缓存是定时炸弹
4. **弱引用**：用 weakref 避免循环引用

## 工程对照（≥100 字硬约束）

内存泄漏在长期运行的服务中危害大：

1. **Web 服务**：长期运行，泄漏累积
2. **定时任务**：周期性任务可能累积
3. **缓存**：用 @lru_cache 限制大小
4. **连接池**：资源未释放
5. **事件监听**：未移除的监听器
6. **调试工具**：objgraph 可视化对象

学完本卡能动手的事：用 tracemalloc 分析自己的脚本，找出潜在泄漏。
