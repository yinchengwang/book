# python-profiling 学习笔记

## 概念地图

Python 性能分析工具：

- **timeit**：精确测量小代码片段执行时间
- **cProfile**：统计每个函数的调用次数和时间
- **tracemalloc**：追踪内存分配
- **memory_profiler**：逐行内存分析

## 性能优化步骤

1. **测量**：先用 profiling 工具找到瓶颈
2. **分析**：理解瓶颈原因
3. **优化**：优化热点代码
4. **验证**：再次测量确认改进

## 常见优化点

1. **循环优化**：减少循环内计算
2. **列表 vs 生成器**：大数据集用生成器
3. **缓存**：避免重复计算
4. **算法复杂度**：O(n²) → O(n log n)

## 工程对照（≥100 字硬约束）

性能分析在实际项目中应用：

1. **瓶颈定位**：cProfile 找到慢函数
2. **内存泄漏**：tracemalloc 追踪增长
3. **算法选择**：timeit 对比不同实现
4. **代码审查**：提交前做性能回归
5. **Benchmark**：持续监控性能变化
6. **优化验证**：优化后用 profiling 确认
