# python-gc 学习笔记

## 概念地图

Python 垃圾回收机制：

- **引用计数**：每个对象维护引用数，为0时立即回收
- **循环 GC**：检测并回收循环引用的对象
- **gc 模块**：手动控制垃圾回收
- **weakref**：弱引用，不增加引用计数

## Python GC 分代

| 代 | 对象 | 回收频率 |
|----|------|----------|
| 0 | 新对象 | 最高 |
| 1 | 幸存对象 | 中等 |
| 2 | 长期幸存 | 最低 |

## 踩坑记录

1. **循环引用泄漏**：对象间相互引用会延迟回收
2. **__del__ 循环**：有 __del__ 的循环可能不回收
3. **禁用 GC**：禁用后循环引用无法回收

## 工程对照（≥100 字硬约束）

GC 知识在实际项目中的应用：

1. **缓存**：用 weakref 避免缓存泄漏
2. **观察 GC**：gc.get_stats() 监控回收
3. **调试泄漏**：gc.set_debug() 追踪泄漏
4. **性能**：频繁创建对象时适当禁用 GC
5. **循环检测**：理解为什么会发生内存增长
6. **对象池**：复用对象减少 GC 压力

学完本卡能动手的事：用 tracemalloc 分析代码的内存分配模式。
