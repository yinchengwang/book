# python-security 学习笔记

## 概念地图

Python 安全编程要点：

| 威胁 | 防御 |
|------|------|
| SQL 注入 | 参数化查询 |
| XSS | HTML 转义 |
| CSRF | CSRF Token |
| 密码泄漏 | 加盐哈希 |
| 会话劫持 | HTTPS + HttpOnly |

## 常见安全问题

1. **注入攻击**：用户输入被当作代码执行
2. **敏感信息泄露**：密码、日志中的敏感数据
3. **弱加密**：使用不安全的算法
4. **依赖漏洞**：第三方库漏洞

## 安全原则

1. **最小权限**：只授予必要权限
2. **输入验证**：验证所有用户输入
3. **输出编码**：防止 XSS
4. **加密存储**：密码用强哈希

## 工程对照（≥100 字硬约束）

安全在生产环境中至关重要：

1. **密码存储**：永远不要明文存储密码
2. **参数化查询**：防止 SQL 注入
3. **依赖审计**：定期更新依赖
4. **敏感信息**：不在日志中记录
5. **HTTPS**：所有通信加密
6. **密钥管理**：不要硬编码密钥

学完本卡能动手的事：检查项目中的用户输入处理，确保使用参数化查询。
