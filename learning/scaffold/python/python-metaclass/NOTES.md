# python-metaclass 学习笔记

## 概念地图

Python 元类是"类的类"：

- **type**：Python 内置元类，所有类的默认类型
- **__new__**：在类创建时调用，控制类的生成过程
- **__init__**：在类创建后调用，初始化类属性
- **metaclass=**：在 class 语句中指定使用的元类

## 元类 vs 装饰器

| 特性 | 元类 | 类装饰器 |
|------|------|----------|
| 作用对象 | 类 | 类实例 |
| 执行时机 | 类定义时 | 类创建后 |
| 灵活性 | 更高 | 较简单 |

## 元类应用场景

1. **ORM**：自动收集字段，生成表结构
2. **注册表**：自动注册插件/模块
3. **API 路由**：Flask/Django 用元类注册路由
4. **序列化**：自动生成序列化/反序列化方法

## 踩坑记录

1. **单例模式**：可用元类实现线程安全的单例
2. **多重继承**：元类可能产生冲突
3. **复杂度**：大多数场景装饰器更简单

## 工程对照（≥100 字硬约束）

元类在 Python 主流框架中广泛应用：

1. **Django ORM**：Model 基类使用元类收集字段
2. **SQLAlchemy**：Declarative 系统用元类定义映射
3. **Flask**：app.add_url_rule 用装饰器，内部也用元类
4. **attrs/ dataclasses**：自动生成 __init__ 等方法
5. **注册模式**：元类自动注册插件，保持代码整洁
6. **序列化**：元类自动生成 to_dict/from_dict
