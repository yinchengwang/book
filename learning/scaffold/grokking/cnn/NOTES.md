# 卷积神经网络 · 学习笔记

## 核心概念

- **卷积操作**：卷积核在输入上滑动并计算点积，提取局部特征（边缘/纹理/形状）
- **池化**：降采样操作（最大池化取最大值、平均池化取均值），减小特征图尺寸，增加平移不变性
- **感受野**：输出特征图上每个像素对应输入图像的区域大小，层数越深感受野越大
- **特征层次**：浅层提取低级特征（边缘/角点），深层组合成高级语义特征

## 工程对照

CNN 的层次化特征提取与本仓库的存储引擎架构高度对应：Buffer Pool 层（类似卷积的底层特征——管理物理页面）、Access Method 层（类似中层特征——提供 Relation 抽象和扫描接口）、SQL 执行器层（类似高层语义——处理查询优化和执行计划）。这种逐层抽象的设计模式在大型软件系统中极其常见。本仓库的 HNSW 索引也在不同层级维护不同粒度的图：顶层图连接稀疏、跳距大（类似 CNN 的深层小特征图，俯瞰全局），底层图连接密集、跳距小（类似 CNN 的浅层大特征图，保留细节）。卷积核的局部连接与参数共享思想（每个核在整个输入上滑动）与本仓库 Buffer Pool 的 Hash 表查找有类比——Hash 函数将页面 ID 映射到固定槽位，同类页面共享同一个"处理逻辑"。
