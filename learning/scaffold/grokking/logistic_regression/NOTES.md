# 逻辑回归 · 学习笔记

## 核心概念

- **Sigmoid 函数**：将实数映射到 (0,1) 概率区间，σ(z) = 1 / (1 + e⁻ᶻ)
- **交叉熵损失**：衡量预测概率分布与真实分布的差异，逻辑回归的标准损失函数
- **决策边界**：逻辑回归在特征空间中的线性分界面，由权重向量和偏置决定
- **梯度下降训练**：最小化交叉熵损失，逐步更新权重参数
- **概率输出**：逻辑回归输出的是概率值，通过阈值（默认 0.5）转换为分类结果

## 工程对照

逻辑回归的决策边界概念在本仓库的存储引擎中有直接的工程对应。在 `engineering/src/db/storage/` 中，BTree 索引（`btreeam.c`）通过键值比较在数据空间中划出搜索范围——每次查找就是一次二分决策。逻辑回归的线性决策边界 w·x + b = 0 与 BTree 索引中每个节点的键值划分（key ≤ threshold vs key > threshold）在本质上共享同一个数学模型：都是通过一个线性阈值将空间一分为二。区别在于逻辑回归的权重是学习得到的，而 BTree 的划分点是由数据分布动态维护的。

交叉熵损失函数在数据库查询优化中也有深远对应。`engineering/src/db/core/query_optimizer.c` 中的查询计划代价估算，本质上也是在最小化「预测代价与实际代价的差异」——类似交叉熵损失。CBO（Cost-Based Optimizer）使用统计信息（表大小、选择率、数据分布）来估算各查询计划的代价，选择一个代价最小的。当统计信息过时或不准确时，优化器做出错误选择——这就如同交叉熵损失中预测概率严重偏离真实标签时的「高损失」场景。逻辑回归训练中通过梯度下降更新权重来降低损失，数据库优化器则通过 ANALYZE 命令刷新统计信息来提高估算准确度。
