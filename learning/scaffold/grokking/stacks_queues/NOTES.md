# 栈与队列

## 概述

栈和队列是两种受限的线性表，栈限制为一端插入/删除（LIFO），队列限制为一端插入另一端删除（FIFO）。它们不是基础数据结构，而是编程抽象——现实中处处都有"后进先出"和"先进先出"的场景。

## 核心题型

### 括号匹配 (LeetCode #20)

**问题**: 判断括号字符串中的括号是否正确闭合。

**解法要点**: 遇到左括号入栈，遇到右括号检查栈顶是否匹配。最后栈必须为空。

**工程应用**: 编译器在解析表达式和代码块时用栈检查大括号匹配；XML/JSON 解析器用栈验证标签/括号的嵌套层级。

### 最小栈 (LeetCode #155)

**问题**: 设计一个支持 push, pop, top, get_min 的栈，所有操作 O(1)。

**解法要点**: 使用"辅助栈"或"差值法"。辅助栈在每个元素入栈时同步记录当前最小值，pop 时同步出栈，get_min 直接取辅助栈顶。

**工程应用**: 光标的撤销/重做（Undo/Redo）操作中，记录每个状态下的快照最小值。

### 约瑟夫环

**问题**: n 个人围成一圈报数，报到 k 的人出列，求最后幸存者的编号。

**解法对比**:

| 方法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 队列模拟 | O(n*k) | O(n) | 直观 |
| 数学递推 | O(n) | O(1) | f(n,k)=(f(n-1,k)+k)%n |

**工程应用**: 操作系统的进程调度（时间片轮转）、分布式系统的资源分配、负载均衡器的轮询。

## 工程对照

**栈 vs 队列 —— 两种调度策略的工程权衡**:

| 维度 | 栈 (LIFO) | 队列 (FIFO) |
|------|-----------|-------------|
| 访问顺序 | 后进先出 | 先进先出 |
| 公平性 | 不公平（新任务强占） | 公平（按到达顺序） |
| 典型场景 | 函数调用、括号匹配、DFS | 任务调度、BFS、缓冲区 |
| 内存局部性 | 好（栈顶附近） | 一般 |

**实际工程选型场景**:

1. **数据库事务回滚**: 使用栈结构存储 undo log。当需要回滚时，从上到下逐层撤销，LIFO 完美适配"倒叙撤销"的需求。如果有事务 A→B→C，回滚时必须以 C→B→A 的顺序执行。

2. **函数调用栈 vs 消息队列**: 函数调用用栈（返回地址压栈），消息队列用队列（先进先出保证处理顺序）。这是因为函数调用天然具有嵌套关系（后调用的先返回），而消息处理通常期待公平调度。

3. **操作系统的中断处理**: 中断嵌套使用栈保存上下文——后面的中断先处理，这是 LIFO 在底层系统中的体现。但进程调度使用队列——就绪队列保证每个进程公平获得 CPU 时间片，这是 FIFO 的体现。

4. **Web 服务器的请求处理**: Nginx 的工作队列使用 FIFO 保证客户请求的公平性。但对于高优先级的请求，可能使用优先级队列（堆）来"插队"——这说明纯 FIFO 在实际系统中也经常被扩展。

5. **图遍历**: 深度优先搜索用栈（隐式或显式），广度优先搜索用队列。栈对应递归的自然展开（函数调用栈），队列对应"逐层扩散"的访问模式。这是栈和队列最经典的算法应用对比。

**关键决策原则**:
- 需要"撤销/恢复" → 栈（回退操作天然反序）
- 需要"公平处理" → 队列（按到达顺序）
- 需要"优先级调度" → 优先级队列/堆
- 实际工程中，环形缓冲区（Ring Buffer）是队列最常用的实现，因为避免了内存的动态分配

## 复杂度总结

| 题型 | 时间复杂度 | 空间复杂度 | 技巧 |
|------|------------|------------|------|
| 括号匹配 | O(n) | O(n) | 栈匹配 |
| 最小栈 | O(1) 每操作 | O(n) | 辅助栈 |
| 约瑟夫环（队列） | O(n*k) | O(n) | 队列模拟 |
| 约瑟夫环（数学） | O(n) | O(1) | 递推公式 |

## 面试要点

1. 栈适合处理"嵌套"结构的问题（括号匹配、表达式求值、DFS）
2. 队列适合处理"顺序"结构的问题（BFS、任务调度、流水线）
3. 双端队列（Deque）是栈和队列的结合体，在滑动窗口问题中非常有用
4. 单调栈/单调队列是栈和队列的进阶用法，用于解决"下一个更大/更小元素"类问题
