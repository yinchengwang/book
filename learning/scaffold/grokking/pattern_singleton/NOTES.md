# 单例模式 学习笔记

## 核心概念

- **单例模式 (Singleton)**: 确保类只有一个实例, 并提供全局访问点
- **饿汉式**: 类加载时创建实例, 天生线程安全
- **懒汉式**: 首次使用时创建, 需要线程安全保护
- **双重检查锁定 (DCL)**: 先检查再同步, 减少锁竞争

## 关键要点

| 实现方式 | 线程安全 | 延迟加载 | 性能 |
|----------|----------|----------|------|
| 饿汉式 | ✅ | ❌ | 高 |
| 懒汉式 + synchronized | ✅ | ✅ | 低 |
| 双重检查 (DCL) | ✅ | ✅ | 高 |
| 静态内部类 | ✅ | ✅ | 高 |

## 工程对照

单例模式在数据库系统中广泛使用。`engineering/src/db/core/guc.c` 中的 GUC (Grand Unified Configuration) 系统就是一个典型的单例应用——整个数据库只有一个配置管理器实例, 所有模块通过 `guc_get()` / `guc_set()` 访问配置参数。`engineering/src/db/buf/bufmgr.c` 中的 Buffer Pool 也可以视为一个单例——数据库进程只维护一个 shared_buffers 缓冲池实例。`engineering/src/db/core/initdb.c` 中的数据库集群初始化过程也只应执行一次, 对应了单例的"唯一性"保障。

## 面试要点

1. C++ 11 后可以用 `static local` 实现线程安全的懒汉单例 (Meyer's Singleton)
2. 单例的典型问题: 难以测试、违反单一职责
3. 现代框架多用 DI (依赖注入) 替代单例
