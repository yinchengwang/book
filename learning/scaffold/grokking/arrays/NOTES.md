# 数组算法

## 概述

数组是最常用的数据结构之一，在算法面试中出现频率极高。

## 核心题型

### 两数之和 (LeetCode #1)

**问题**: 给定数组和目标值，找出两数之和为目标值的两个索引。

**解法对比**:

| 方法 | 时间 | 空间 | 说明 |
|------|------|------|------|
| 暴力枚举 | O(n²) | O(1) | 两层循环 |
| 排序+双指针 | O(n log n) | O(1) | 需处理索引映射 |
| 哈希表 | O(n) | O(n) | 空间换时间 |

**工程应用**: 在电商系统中查找满足满减条件的商品组合、在交易系统中查找汇率兑换组合。

### 旋转数组搜索 (LeetCode #33)

**问题**: 在旋转排序数组中查找目标值。

**关键洞察**: 旋转数组可以看作两个有序子数组的拼接，利用二分查找的特性可以判断目标在哪个区间。

**工程应用**: 在有序数据中快速定位场景，如日志时间范围查询、股票价格区间查询。

### 最大子数组和 (LeetCode #53)

**问题**: 找出连续子数组的最大和。

**Kadane 算法核心**: 当当前和为负时，重新开始比保留更有利。

```c
// 核心逻辑
current_sum = max(current_sum + num, num);
max_sum = max(max_sum, current_sum);
```

**工程应用**:
- 监控系统中的峰值统计
- 金融系统的最大收益计算
- 性能分析中的最大连续上涨区间

## 工程对照

**数据库索引扫描**: 数组的 O(log n) 二分查找对应数据库的 B-Tree 索引扫描。当数据量从 1 万增长到 1 亿时，O(log n) 的查询只需从 14 次比较增加到 27 次，性能稳定。

**缓存淘汰策略**: 最大子数组和的滑动窗口思想可用于 LRU 缓存的热点数据分析，识别持续活跃的数据项进行保序。

**流式数据处理**: 在实时数据流中计算滑动窗口指标，如最近 5 分钟的平均值、最大值等，滑动窗口算法是流处理框架(Flink/Spark Streaming)的基础。

## 复杂度总结

| 题型 | 时间复杂度 | 空间复杂度 | 技巧 |
|------|------------|------------|------|
| 两数之和 | O(n) | O(n) | 哈希表 |
| 旋转数组 | O(log n) | O(1) | 二分边界判断 |
| 最大子数组 | O(n) | O(1) | Kadane 算法 |

## 面试要点

1. 先考虑暴力解法，确认复杂度要求
2. 利用数组有序性考虑二分查找
3. 滑动窗口适合连续子数组问题
4. 哈希表是 O(n) 解法的利器
