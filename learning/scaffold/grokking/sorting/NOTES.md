# 排序算法实战

## 工程对照

### 排序算法在工程中的选择

工程实践中选择排序算法需要综合考虑数据规模、数据分布、内存限制和稳定性要求：

| 数据场景 | 推荐算法 | 原因 |
|----------|----------|------|
| 通用内存排序（n < 10^6） | 快速排序/归并排序 | 平均 O(n log n)，Cache 友好 |
| 外部排序（数据量 > 内存） | 归并排序 | 外排序标准算法，适合磁盘 IO |
| 已知范围整数排序 | 计数排序/桶排序 | O(n) 时间复杂度 |
| 找 Top K 元素 | 快速选择/堆 | 无需完全排序，O(n) 或 O(n log k) |
| 稳定性要求严格 | 归并排序 | 稳定的 O(n log n) 算法 |

**工程经验**：在 PostgreSQL 存储引擎中，排序操作用于 BTree 索引构建、查询结果排序（ORDER BY）、去重（DISTINCT）等场景。PostgreSQL 使用归并排序和快速排序的混合策略，通过 `work_mem` 参数控制内存排序阈值。

### 工程对照总结

排序是数据库系统的核心操作，PostgreSQL 的排序实现非常值得学习：
1. **Tuplesort**：统一的排序接口，支持 Heap Sort、Quick Sort、External Merge Sort
2. **Index Sort**：利用已有索引避免重排
3. **Sort Memory**：通过 `work_mem` 控制内存使用
4. **Top-N Heap**：对于只需要 Top K 的查询，使用堆而非完全排序

## 重点

### 快速选择的正确性分析

快速选择算法基于分区的思想，其正确性依赖于：
1. **分区函数**将数组分为两部分：大于基准的元素和小于等于基准的元素
2. **递归终止条件**：当第 K 大的元素恰好在分区点时返回
3. **递归方向选择**：根据 K 与分区点位置的关系决定向左还是向右递归

关键不变式：
```
partition 后，pivot_idx 位置的元素是数组中第 (pivot_idx - left + 1) 大的元素
```

### 桶排序的适用条件

桶排序的高性能依赖于数据分布均匀：
- **均匀分布**：每个桶的元素数量接近，时间复杂度接近 O(n)
- **不均匀分布**：最坏情况退化为 O(n^2)

工程中桶排序常用于：
1. **浮点数排序**：范围已知时按小数部分分桶
2. **字符串排序**：按前缀分桶（基数排序的变体）
3. **分布式系统**：按 Key 的哈希值分桶

### 区间合并的边界处理

区间合并需要注意的边界情况：
1. **完全重叠**：`[1,5]` 和 `[2,3]` → `[1,5]`
2. **首尾相连**：`[1,3]` 和 `[3,5]` → `[1,5]`（注意 3 == 3 是重叠的）
3. **空输入**：返回空数组
4. **单区间**：直接返回
