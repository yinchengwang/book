# 适配器模式 学习笔记

## 核心概念

- **适配器模式 (Adapter)**: 使不兼容的接口能够协同工作
- **类适配器**: 通过多继承实现 (继承目标接口 + 继承被适配者)
- **对象适配器**: 通过组合实现 (实现目标接口 + 持有被适配者引用)

## 类适配器 vs 对象适配器

| 维度 | 类适配器 | 对象适配器 |
|------|----------|------------|
| 实现方式 | 继承 | 组合 |
| 灵活性 | 低 (编译时固定) | 高 (运行时切换) |
| 语言限制 | 需要多继承 | 任何语言 |
| 推荐度 | 低 | 高 (优先组合) |

## 工程对照

接口兼容性在 `engineering/` 轨中有多处实践。`engineering/include/db/mm_storage.h` 定义的多模态存储引擎接口本质上是适配器模式的应用——每种存储引擎 (KV/Vector/Graph/Timeseries) 都需要实现统一的 `storage_engine_t` 接口，将各自的内部 API 适配到标准框架上。`engineering/src/db/buf/bufmgr.c` 中的 Buffer Pool 接口 (`buffer_get_page`, `buffer_unpin_page` 等) 也是将物理磁盘读写适配为逻辑页面操作的桥梁。

## 面试要点

1. 适配器 vs 桥接: 适配器用于后期兼容, 桥接用于前期设计
2. 优先使用对象适配器 (组合优于继承)
3. 适配器可以包装第三方库, 降低更换成本
