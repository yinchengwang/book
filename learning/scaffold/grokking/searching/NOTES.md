# 二分查找工程对照

## 实际应用场景

二分查找思想在工程中无处不在：

- **数据库索引（B+树节点内二分）**：B+树的内部节点按 key 有序排列，在节点内通过二分查找确定下一步走向的子树。该查找的 I/O 代价远低于全表扫描。
- **内存分配器（buddy system）**：内核 Buddy 分配器在空闲区间列表中做二分搜索，快速找到大小合适的连续物理内存块。
- **日志系统（二分查时间戳）**：分布式系统的 WAL（Write-Ahead Log）中，利用时间戳递增特性，在日志段上做二分定位，快速回放到指定时间点。
- **版本号查找（CAS 乐观锁）**：在多版本并发控制（MVCC）中，事务通过二分查找 Version Chain，快速定位对本条记录可见的快照版本。
- **网络路由（CIDR 前缀匹配）**：路由表用 Trie 或二分查 LPM（最长前缀匹配），在 O(log n) 内找到下一跳地址。
- **CPU 微码（Branch Prediction）**：硬件层面，分支目标缓冲区（BTB）通过二分查 tag 预测下一条指令地址。

这些场景虽然表现形式各异，本质都是 **通过有序性消除无效搜索空间**，这也是计算机科学中"分治"思想的底层逻辑。
