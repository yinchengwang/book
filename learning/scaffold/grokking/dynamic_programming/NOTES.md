# DP 套路总结

## 解题四步法

1. **定义状态** —— 明确 dp[i] 或 dp[i][j] 的含义。例如「爬楼梯」dp[i] 表示走到第 i 阶的方法数；「编辑距离」dp[i][j] 表示 word1 前 i 个字符转换成 word2 前 j 个字符的最小步数。
2. **找出递推公式** —— 思考从上一个状态如何转移到当前状态。这是 DP 的核心，也是最难的一步。
3. **初始化边界** —— 处理 dp[0]、dp[0][...]、dp[...][0] 等基本情况。边界错误会导致整个递推偏移。
4. **确定遍历顺序** —— 由递推公式决定：一维常正向，二维常逐行或逐列。

## 常见 DP 类型
- **线性 DP（一维数组）**：爬楼梯、最大子数组和、打家劫舍
- **区间 DP（二维区间）**：最长回文子串、矩阵链乘
- **背包 DP（容量约束）**：0-1 背包、完全背包
- **树形 DP（树上递归）**：二叉树最大路径和、树形打家劫舍
- **状态机 DP（状态转移）**：股票买卖系列、多阶段决策
- **数位 DP（按位递推）**：数字范围内的计数问题

工程中的 DP 例子：Viterbi 算法（HMM 解码）、Needleman-Wunsch（基因序列比对）、CTCP 协议中的拥塞窗口自适应（AIMD）、Linux 内核的 CFQ I/O 调度（根据历史 IO 模式预测下一请求）。
