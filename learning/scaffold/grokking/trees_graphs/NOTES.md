# 树图技巧笔记

## 标题

二叉树遍历与搜索技巧

## 核心技巧

### 1. BFS 层序遍历

使用队列实现，按层级访问节点。适合求最大深度、层平均值等。

### 2. 对称性判断

递归比较左子树和右子树是否互为镜像。关键洞察：左左对右右，左右对右左。

### 3. 路径总和

从根到叶的路径和判断。每次递归减去当前节点值，叶子节点时判断是否归零。

## 工程对照

在真实工程项目中，树结构无处不在：

1. **文件系统**
   - 目录树：每个目录是其父目录的子节点
   - 路径解析：`/usr/local/bin` 本质是树遍历
   - Linux VFS（虚拟文件系统）使用 dentry 结构体组织成树
   - Git 的对象存储也是树结构（commit 指向 tree）

2. **XML/JSON 解析**
   - DOM 树：每个标签/对象是一个节点
   - 解析器内部使用递归下降解析
   - XPath/XQuery 本质是对 DOM 树的查询

3. **数据库索引**
   - B-Tree/B+Tree 是数据库索引的核心结构
   - PostgreSQL 的 pg_am 系统表管理各种访问方法
   - 范围查询、排序都依赖树结构

4. **网络路由**
   - 路由表按前缀组织成 Patricia Trie
   - BGP 路由选择使用路径向量算法

5. **编译原理**
   - 抽象语法树（AST）是编译的核心数据结构
   - 语义分析依赖对 AST 的遍历
   - LLVM IR 也是树状/图状结构

## 重点掌握

- **递归思维**：树的问题大多可以用递归解决
- **状态传递**：路径和问题中目标值作为状态传递
- **终止条件**：叶子节点的识别和处理
