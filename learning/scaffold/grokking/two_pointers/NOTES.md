# 双指针技巧

## 工程对照

双指针是数据库查询执行器和网络协议解析中的核心范式。在排序归并连接（Sort-Merge Join）中，两个有序表使用双指针并行扫描完成连接操作，时间复杂度 O(N+M)。在内存池和缓冲区管理中，双指针用于管理空闲链表的分配与回收。此外，环形缓冲区（Ring Buffer）的生产者-消费者模型本质也是双指针的应用。在 Redis 的列表类型中，通过头尾双指针实现了 O(1) 的 LPUSH/RPUSH 和 LPOP/RPOP。数据库 B+ 树的叶子节点遍历也采用了类似的指针推进模式来执行范围扫描。

## 重点

- **性能优化**：双指针的关键在于利用数据的有序性或单调性，将嵌套循环降维为线性扫描。排序 + 双指针是面试中最常见的优化套路。
- **空间复杂度**：双指针通常只使用 O(1) 额外空间（如接雨水和环形链表检测），在内存受限的场景下尤为珍贵。
- **边界处理**：双指针的核心难点在于指针移动策略和边界条件控制。三数之和中的去重逻辑、接雨水中 left_max/right_max 的更新时机、环形链表的步长控制，都是常见的易错点。
