# 查询优化 · 学习笔记

## 核心概念

1. **查询计划（Query Plan）**：数据库优化器将 SQL 转换为执行算子树的过程
2. **索引覆盖（Covering Index）**：查询列全在索引中时，只需读索引页，无需回表
3. **慢查询优化**：避免 LIKE %pattern%、避免函数包裹列、合理使用复合索引
4. **复合索引最佳实践**：等值条件列放前面，范围条件列放后面

## 常见索引失效场景

| 场景 | 反例 | 正例 |
|------|------|------|
| LIKE 模糊前缀 | `LIKE '%abc'` | `LIKE 'abc%'` |
| 隐式类型转换 | `WHERE id = '123'`（字符串查数字） | 类型匹配 |
| 函数包裹列 | `WHERE YEAR(date) = 2026` | `WHERE date >= '2026-01-01'` |
| 跳过了最左列 | 复合索引(a,b)，`WHERE b=?` | 使用 a 条件 |

## 工程对照

本项目的查询优化实现在 `engineering/src/db/sql/optimizer.c`。优化器对 SQL
解析器生成的 AST 进行分析，决定是否使用索引：遍历 WHERE 条件的列，与系统表
`pg_index` 中的索引定义匹配。若匹配则生成 IndexScan 节点，否则回退为 SeqScan。
与本文 SQLite EXPLAIN 不同的是，工程实现需要自己管理查询计划的内存结构
和代价估算。
