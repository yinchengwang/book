# 梯度提升 · 学习笔记

## 核心概念

- **Boosting**：串行训练弱学习器，每个新学习器纠正前一个的残差
- **残差拟合**：GBDT 的每棵树拟合的是前一轮的预测残差（负梯度方向）
- **学习率（Shrinkage）**：缩小每棵树的贡献，用小步多走的方式减少过拟合
- **正则化**：子采样（subsample）、列采样、学习率衰减、早停等

## 工程对照

Boosting 的逐步纠正思想与本仓库 db 存储引擎的 Buffer Pool 刷脏策略有类比性：Buffer Pool 使用 Clock-Sweep 置换算法逐步淘汰冷页面，每一次 Clock 指针移动相当于 Boosting 中的"一步"，以前一轮访问位（reference bit）为依据决定淘汰哪些页面。若淘汰策略过于激进（类似高学习率），会导致频繁的页面震荡（颠簸）；若过于保守（低学习率），则页面长时间占用 buffer 导致利用率下降。这与 GBDT 中学习率的选择——需要在收敛速度和过拟合之间权衡——是类似的调参问题。本仓库 bufmgr.c 中的 Clock-Sweep 实现提供了 `clock_hand` 访问模式和脏页回写调度，类似于 Boosting 中迭代式的残差纠正过程。
