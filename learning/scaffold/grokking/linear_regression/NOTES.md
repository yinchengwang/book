# 线性回归 · 学习笔记

## 核心概念

- **一元线性回归**：一个特征，拟合直线 y = wx + b，最小二乘法求闭式解
- **多元线性回归**：多个特征，y = w₁x₁ + w₂x₂ + ... + b，矩阵形式求解
- **梯度下降**：迭代优化算法，沿损失函数梯度方向更新参数，学习率控制步长
- **评估指标**：MSE（均方误差）、RMSE（均方根误差）、MAE（平均绝对误差）、R²（决定系数）
- **闭式解 vs 迭代解**：闭式解一步到位但 O(n³) 复杂度，梯度下降适合大规模数据

## 工程对照

线性回归中的梯度下降优化与本仓库 `algo` 库（`engineering/src/algo/`）中的数值计算有深刻的联系。梯度下降本质上是一个迭代优化问题：每次沿着梯度负方向更新参数，逐步逼近最优解。这与 `algo` 库中的排序算法（如 QuickSort 的递归分治）和二分查找（每次缩小一半搜索空间）共享相似的「迭代收敛」思想——只不过排序是在元素序列上收敛到有序状态，而梯度下降是在参数空间上收敛到最优值。

具体来说，梯度下降的学习率参数类似于 `algo` 库排序算法中的比较函数——学习率控制了「每一步走多远」，类似比较函数决定了「元素的顺序」。学习率过大导致震荡不收敛（类比排序中的不稳定排序），学习率过小导致收敛慢（类比低效 O(n²) 排序）。本仓库 `algo` 库中实现了多种排序、搜索和数值算法，每种算法都有其复杂度特征和适用场景，这与回归模型选择（一元 vs 多元、闭式解 vs 迭代解）的权衡取舍本质上一致。

另一个值得关注的工程对照是 R² 决定系数的含义：它衡量了模型解释了多少数据方差。在本仓库的 benchmark 测试中（`engineering/test/` 的性能基准测试），我们同样需要「解释」性能波动——是缓存命中率（类似 R² 中的可解释方差）还是系统负载（类似残差方差）导致了性能变化？这种方差分解思维是理解和优化系统性能的关键。
