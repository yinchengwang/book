# 堆技巧笔记

## 标题

堆结构与 Top-K 问题

## 核心技巧

### 1. 堆的基本操作

- **push**: 将元素添加到堆尾，然后上浮调整
- **pop**: 移除堆顶，用堆尾替换，然后下沉调整
- **peek**: 获取堆顶元素但不删除

### 2. Top-K 的堆解法

对于海量数据找 Top-K，只维护一个大小为 K 的小顶堆：
- 堆满后，较大的元素替换堆顶并调整
- 时间复杂度 O(n log k)，空间 O(k)

### 3. 双堆求中位数

用最大堆存较小的一半，用最小堆存较大的一半：
- 两堆平衡时取平均，不平衡时取元素多的堆顶
- 每次插入 O(log n)，求中位数 O(1)

## 工程对照

在真实工程项目中，堆结构有广泛的应用：

1. **进程调度**
   - Linux CFS 调度器用红黑树（平衡二叉搜索树）管理进程
   - 实时系统用优先级堆实现优先级调度
   - Windows 消息队列按优先级组织

2. **内存分配器**
   - glibc 的 ptmalloc 使用 bins 管理空闲块
   - jemalloc 用多层级堆管理不同大小的内存块
   - 伙伴系统用二叉树管理内存页

3. **外部排序与海量数据处理**
   - 当数据量无法全部放入内存时，用 K 路归并
   - MapReduce 的 Shuffle 阶段用堆做归并排序
   - 数据库的 Sort-Merge Join 使用多路归并

4. **优先队列**
   - Dijkstra 最短路径算法用堆优化
   - Prim 最小生成树算法
   - A* 寻路算法的 open set
   - 任务调度系统的事件队列

5. **统计与监控**
   - 流式数据中持续维护 Top-K 指标
   - 滑动窗口中的中位数查询
   - Prometheus 等监控系统用堆实现百分位数

## 重点掌握

- **堆的数组实现**：完全二叉树的顺序存储，索引关系
- **上浮下沉**：堆调整的核心操作
- **堆 vs 二叉搜索树**：堆保证堆序性，不保证顺序性
- **变体**：双堆、多堆、斐波那契堆等高级变体
