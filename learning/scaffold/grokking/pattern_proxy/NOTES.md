# 代理模式 学习笔记

## 核心概念

- **代理模式 (Proxy)**: 为另一个对象提供替身或占位符以控制访问
- **虚代理 (Virtual Proxy)**: 延迟加载大对象
- **保护代理 (Protection Proxy)**: 控制访问权限
- **远程代理 (Remote Proxy)**: 封装远程调用
- **缓存代理 (Cache Proxy)**: 缓存结果, 减少计算

## 代理模式分类

| 类型 | 目的 | 典型场景 |
|------|------|----------|
| 虚代理 | 延迟加载, 节省资源 | 大图片/大文档加载 |
| 保护代理 | 权限控制 | 文档/API 访问控制 |
| 远程代理 | 隐藏远程调用细节 | RPC/gRPC 客户端 |
| 缓存代理 | 减少重复计算 | 计算密集型结果缓存 |

## 工程对照

间接访问控制在 `engineering/` 轨的数据库内核中有丰富体现。`engineering/src/db/buf/bufmgr.c` 的 Buffer Pool 可以理解为一个"缓存代理"——应用程序通过 Buffer Pool 访问磁盘页面, Buffer Pool 负责在内存中缓存热点页面, 减少磁盘 I/O。`engineering/include/db/lock.h` 中的锁管理器相当于"保护代理"——事务在访问数据之前必须先获得锁授权，通过锁来控制并发访问。`engineering/include/db/mm_storage.h` 的多模态存储引擎选择器则类似于"远程代理"——上层不需要关心底层使用的是哪种存储引擎。

## 面试要点

1. 代理 vs 装饰器: 代理控制访问, 装饰器增强功能
2. 代理 vs 适配器: 代理保持接口不变, 适配器改变接口
3. C++ 中可以用 operator-> 和 operator* 实现智能指针代理
