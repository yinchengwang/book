# 位运算技巧

## 工程对照

位运算是嵌入式系统、数据库引擎和操作系统内核中的基础技能。在实际工程中，位掩码被广泛用于权限管理系统（如 Linux 文件权限的 rwx 位表示）、状态压缩（如数据库事务中多个状态的位图组合）、以及高性能标志位操作（如 epoll 事件掩码）。此外，数据库存储引擎中广泛使用位图来标记页面空闲空间、跟踪缓冲区池状态，Redis 的 SDS 字符串也使用位操作来管理内存分配。在 OLAP 系统中，位图索引（Bitmap Index）通过每个比特位对应一行记录，大幅提升范围查询和聚合查询的效率，这些都是位运算从基础技巧到生产落地的典型范例。

## 重点

- **性能优化**：位运算的指令周期通常为 1 个 CPU 周期，远快于乘除法和分支判断。在设计核心路径的数据结构时，优先考虑用位运算替代模运算和条件分支。
- **权限管理**：将多个权限编码到一个整型字段中，通过位掩码实现 O(1) 的增删查操作，这是 Unix 文件权限和 RBAC 系统的经典模式。
- **状态压缩**：位图（bitmap）将一组布尔标志压缩到单个字中，在数据库缓冲池、网络协议头、内存分配器中大量使用。
