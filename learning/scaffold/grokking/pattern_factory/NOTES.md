# 工厂模式 学习笔记

## 核心概念

- **简单工厂 (Simple Factory)**: 一个工厂类集中创建逻辑
- **工厂方法 (Factory Method)**: 父类定义创建接口, 子类决定实例化哪个类
- **抽象工厂 (Abstract Factory)**: 创建相关或依赖的产品家族

## 三种工厂对比

| 模式 | 复杂度 | 扩展方式 | 典型场景 |
|------|--------|----------|----------|
| 简单工厂 | 低 | 修改工厂类 | 创建逻辑简单固定 |
| 工厂方法 | 中 | 新增工厂子类 | 框架扩展点 |
| 抽象工厂 | 高 | 新增工厂产品族 | 跨平台 UI 组件 |

## 工程对照

对象创建的解耦在 `engineering/` 轨中直接通过 C 语言的结构体和函数指针实现。`engineering/include/db/storage_engine.h` 中定义的 `storage_engine_t` 结构体通过注册机制实现了工厂模式的变体——各存储引擎 (KV/Vector/Graph/Document) 通过 `register_engine()` 函数注册自己的创建函数和操作函数指针。C 语言虽然没有类和继承, 但通过结构体 + 函数指针 + 注册机制同样可以实现工厂模式的解耦效果。

## 面试要点

1. 简单工厂不是 GoF 23 模式之一 (它是编程惯用法)
2. 工厂方法和抽象工厂的区别: 前者创建单一产品, 后者创建产品家族
3. Spring IoC 容器本质上就是一个高级的抽象工厂
