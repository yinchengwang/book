# 内存池 (Memory Pool)

本目录演示三种常见的内存分配优化技术。

## 目录内容

| 文件 | 说明 |
|------|------|
| `main.cpp` | 演示代码，包含三种内存池实现 |

## 三种内存池

### 1. 定长内存池 (FixedSizePool)

适用于对象大小固定的场景，如游戏中的粒子系统、网络包的缓冲池。

- 预先分配固定大小的内存块
- 用链表管理空闲块
- 分配/归还 O(1)

### 2. 对象池 (ObjectPool)

适用于对象创建销毁频繁的场景，如数据库连接池、线程池。

- 模板类，可复用任意类型对象
- 预先创建对象实例
- 归还时只放回池中，不真正释放

### 3. 内存竞技场 (Arena)

适用于批处理场景，如解析器、编译器。

- 一次分配，批量释放
- 支持对齐分配
- Reset() 后复用整块内存

## 适用场景

| 类型 | 适用场景 | 不适用场景 |
|------|----------|------------|
| FixedSizePool | 大小固定的高频分配 | 大小不一的场景 |
| ObjectPool | 复杂对象频繁创建销毁 | 对象有复杂依赖关系 |
| Arena | 批处理/临时内存 | 需要单独释放某块内存 |
