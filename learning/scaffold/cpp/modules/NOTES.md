// NOTES.md — C++20 模块工程对照
//
// 模块化设计思想在项目中的应用：
//
// 1. 头文件 vs 模块
//    - 传统：`#include` 每次都完整展开，编译慢
//    - 模块：`import` 增量加载，编译快
//
// 2. 项目中的模块化实践
//    - `engineering/src/db/` 按功能模块划分
//      * storage/ — 存储层
//      * catalog/ — 目录层
//      * index/   — 索引层
//    - 头文件作为模块接口，源文件作为实现
//
// 3. 未来演进方向
//    - C++20 模块成熟后可替代头文件
//    - 编译时间可显著缩短（尤其是增量编译）
//    - 更好的编译防火墙（避免泄漏宏定义）
//
// 4. 过渡策略
//    - 现有项目保持头文件方式
//    - 新模块可尝试 .ixx 接口文件
//    - 关注编译器对模块的支持进展
//
// 参考：`engineering/include/db/` 中的头文件设计。
