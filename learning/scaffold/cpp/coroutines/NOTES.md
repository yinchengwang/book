// NOTES.md — C++20 协程工程对照
//
// 工程中的异步编程模式对比：
//
// 1. 传统回调模式
//    回调嵌套过深（Callback Hell），代码难维护。
//    适用于简单场景，但不推荐复杂业务。
//
// 2. std::future / std::promise
//    C++11 引入，提供异步计算结果。
//    问题：只能用于单值传递，不支持状态机。
//
// 3. C++20 协程（推荐）
//    - 同步写法，异步执行
//    - 内存占用极低（栈帧保存而非完整线程）
//    - 适用于 I/O 密集型任务
//
// 4. 第三方库（libuv / Boost.Asio）
//    成熟的异步框架，支持网络/文件 I/O。
//    协程化后可与 C++20 协程结合使用。
//
// 项目中的潜在应用：
// - `engineering/src/db/wal/wal.c` 可考虑用协程简化异步刷盘
// - 网络请求可用协程替代回调链
// - 数据库查询的异步等待
//
// 参考：`engineering/src/db/` 目录中的异步逻辑。
