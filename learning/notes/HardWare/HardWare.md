## cache line
​	计算机内存系统中用于存储和传输数据的最小单位，通常是一组连续字节。它的大小对计算机的性能有显著影响，因为处理器从主存读取数据时，会一次性读取整个Cache Line，而不是单独读取需要的那个字节。这样做可以减少Cache未命中的情况，提高处理器访问数据的速度。

- 大小：通常为2的幂次方。64或者128字节等。
- 作用：作为CPU与主内存之间的高速数据缓冲区。

## cache miss
​	Cache Miss（缓存未命中）是指在计算机系统中，处理器在缓存中无法找到所需数据的情况。这种情况下，处理器需要从更慢的主存中读取数据，这会导致额外的延迟和性能下降
- 原因
  - 系统刚启动或程序首次运行时，缓存中尚未加载所需数据
  - 缓存容量有限，无法容纳所有需要的数据。此时和缓存淘汰策略有关(LRU, MRU等)。

- 影响
  -  操作时延增加：从主存中找比在缓存中找要慢得多，导致访问延迟
  - 性能下降：频繁的未命中会降低系统性能(**L1缓存 → L2缓存 → L3缓存 → 内存 → 磁盘**)。

- 解决方案
  - 增加缓存大小以容纳更多的数据
  - 改变缓存淘汰策略
  - 预取数据：利用算法提前加载可能需要的数据
  - 软件优化：优化算法和数据结构，减少随机数据访问


## L1 cache

​	L1(`Level 1`)缓存是**位于CPU内部**的第一级高速缓存，用于存储指令和数据。L1缓存被进一步分为指令缓存(`I-Cache`)和数据缓存(`D-cache`)，它们分别专门用于存储指令和数据。`I-Cache`存储CPU执行的指令，而`D-cache`存储CPU读取和写入的数据。

​	I-Cache和D-cache的目的是通过提供更快的数据访问速度来减少对主内存的访问。当处理器核心需要执行指令时，它会首先在I-Cache中查找，如果指令已经缓存在I-Cache中，则可以立即执行。同样，当处理器核心需要读取或写入数据时，它会首先在D-cache中查找，如果数据已经缓存在D-cache中，则可以快速访问。
